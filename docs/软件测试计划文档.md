# 软件测试计划文档——基于知识图谱的软件工程课程问答平台

---

## 1 引言

### 1.1 编写目的

本文档是基于知识图谱的软件工程课程问答平台的软件测试计划文档，旨在明确测试的对象、目标、策略、过程和进度安排，为软件测试活动提供指导和依据。通过系统性的测试，验证各功能模块的正确性、稳定性和性能表现，确保软件质量满足用户需求。

### 1.2 背景

a）本测试计划是基于知识图谱的软件工程课程问答平台开发文档的组成部分。

b）基于知识图谱的软件工程课程问答平台是一个面向软件工程课程的教学辅助平台，结合知识图谱、向量搜索和大语言模型技术，提供知识图谱构建与可视化、文档管理、题库练习和学习分析等功能。

c）系统采用前后端分离架构：前端基于Vue 3 + TypeScript，后端基于Go（Gin框架），AI服务基于Python（FastAPI框架），数据存储使用MySQL和Neo4j。

d）执行本测试计划之前确保以下前提条件已满足：

- 前端界面搭建完成，各页面可正常访问
- 后端API接口开发完成，可正常响应请求
- Python AI服务部署完成，可正常调用
- MySQL数据库和Neo4j图数据库安装配置完成
- 测试环境搭建完毕，测试数据准备就绪

### 1.3 定义

| 术语 | 定义 |
|------|------|
| 知识图谱 | 从文档中提取知识点和关系，以图结构存储的知识表示方法 |
| FAISS | Facebook AI Similarity Search，向量相似度搜索库 |
| Neo4j | 图数据库，用于存储知识图谱 |
| Ollama | 本地大语言模型运行框架 |
| JWT | JSON Web Token，用于用户认证的令牌机制 |
| 知识点 | 从文档中提取的实体或概念 |
| 知识关系 | 知识点之间的语义关联 |

## 2 计划

### 2.1 软件说明

基于知识图谱的软件工程课程问答平台包含以下功能模块：

| 模块编号 | 模块名称 | 功能描述 |
|----------|----------|----------|
| M1 | 用户认证模块 | 用户注册、登录、JWT令牌管理 |
| M2 | 文档管理模块 | 文档上传、查询、删除、内容管理 |
| M3 | 知识图谱构建模块 | 从文档中提取知识点和关系，构建知识图谱 |
| M4 | 知识图谱可视化模块 | 以图形方式展示知识图谱结构 |
| M5 | 题库练习模块 | 题目展示、答题提交、成绩评分 |
| M6 | 学习分析模块 | 测试结果分析、薄弱知识点分析 |

### 2.2 测试内容

本测试计划针对以下核心功能进行测试：

**（1）用户认证功能测试**

- 用户注册：输入用户名、密码进行注册
- 用户登录：输入用户名、密码进行登录，获取JWT令牌
- 令牌验证：验证JWT令牌的有效性和过期处理

**（2）文档管理功能测试**

- 文档上传：上传txt格式的课程文档
- 文档查询：查看已上传文档列表
- 文档删除：删除指定文档

**（3）知识图谱构建功能测试**

- 知识图谱构建：选择文档后调用AI服务提取知识点和关系
- 构建结果查询：查看最近一次构建结果
- 知识图谱数据查询：获取知识点和关系列表

**（4）知识图谱可视化功能测试**

- 图谱展示：以节点和连线形式展示知识图谱
- 交互操作：支持缩放、拖拽、点击节点查看详情

**（5）题库练习功能测试**

- 题目展示：随机或按类别展示题目
- 答题提交：选择答案并提交
- 成绩评分：自动评分并显示解析

**（6）学习分析功能测试**

- 测试分析：分析答题正确率和薄弱环节

### 2.3 测试条件

**2.3.1 硬件环境**

| 设备 | 配置要求 |
|------|----------|
| PC终端 | Intel Core i5及以上，8GB内存以上 |
| 服务器 | 用于部署后端服务和AI服务（可与PC终端相同） |

**2.3.2 软件环境**

a）操作系统：Windows 10/11

b）浏览器：Chrome 90+、Edge 90+（前端页面访问）

c）运行环境：

- Go 1.21+（后端服务运行）
- Python 3.11+（AI服务运行）
- Node.js 18+（前端开发和构建）

d）数据库：

- MySQL 8.0（关系数据存储）
- Neo4j 5.x（图数据库，存储知识图谱）

e）AI服务依赖：

- Ollama（本地大语言模型运行，需安装qwen3:8b模型）
- FAISS（向量相似度搜索）

f）网络连接：本地网络连接正常

**2.3.3 测试资料**

a）测试用课程文档：软件工程相关知识文档（txt格式）

b）测试用用户账号：预设的测试用户（用户名：testuser，密码：test123）

c）API接口文档：后端各接口的请求参数和响应格式说明

d）数据库表结构：用户表、文档表、知识点表、知识关系表等

### 2.4 测试资料详细说明

**（1）文档管理模块输入输出**

输入：文档文件（txt格式，包含软件工程知识内容）

输出：文档列表（包含文档ID、标题、创建时间等信息），操作结果提示

**（2）知识图谱构建模块输入输出**

输入：文档ID列表（选择要构建知识图谱的文档）

输出：知识点列表（包含知识点名称、类别、描述），知识关系列表（包含源知识点、目标知识点、关系类型），构建状态提示

**（3）题库练习模块输入输出**

输入：用户选择的答案选项

输出：评分结果（正确/错误），题目解析，得分统计

### 2.5 运行时间的测试

本测试需要测量各核心功能的执行时间，确保系统响应时间满足用户体验要求。

**2.5.1 进度安排**

| 时间 | 任务 |

| 第1天上午 | 编写时间测试代码，明确测试要点，准备测试数据 |
| 第1天下午 | 执行各模块响应时间测试，记录测试过程并分析结果 |
| 第2天上午 | 执行知识图谱构建响应时间测试 |
| 第2天下午 | 汇总测试数据，分析性能瓶颈，编写测试报告 |

**2.5.2 测试资料**

a）测试用文档数据：10篇不同长度的软件工程文档

b）测试用问题数据：20个不同复杂度的软件工程问题

c）性能指标要求：

| 功能 | 响应时间要求 |

## 5 风险与意外

| 风险项 | 可能影响 | 应对措施 |
|--------|----------|----------|
| Ollama服务不可用 | 知识图谱构建功能无法使用 | 检查Ollama服务状态，确保模型已正确加载 |
| Neo4j数据库连接失败 | 知识图谱存储和查询功能异常 | 检查Neo4j服务状态，重启服务 |
| MySQL数据库连接失败 | 所有数据存储功能异常 | 检查MySQL服务状态，验证连接配置 |
| 测试数据不足 | 无法充分覆盖测试场景 | 提前准备充足的测试文档和问题数据 |
| 网络延迟影响 | 响应时间测试结果不准确 | 在本地环境测试，排除网络因素 |
