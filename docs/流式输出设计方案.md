# AI 问答流式输出设计方案

## 1. 概述

本文档描述了 SE 智图问答平台中 AI 问答流式输出功能的设计与实现方案。流式输出允许用户在 AI 生成回答的过程中实时看到内容，而不是等待完整的回答生成后才显示。

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端 Vue  │ ←→  │  Go 后端    │ ←→  │ AnswerService│ ←→  │   Ollama    │
│  (EventSource)│    │  (SSE)     │     │  (Stream)   │     │  (Stream API)│
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 2.2 数据流

```
用户输入问题
    ↓
前端调用 askQuestionStream()
    ↓
POST /api/v1/ask/stream (SSE)
    ↓
Controller 设置 SSE 响应头
    ↓
Service 层调用 AI Client
    ↓
OllamaClient.GenerateStream()
    ↓
POST http://localhost:11434/api/generate (stream: true)
    ↓
逐 token 返回 → channel → SSE Events → 前端实时显示
```

### 3.4 SSE Controller

**文件**: `internal/api/v1/ask/controller.go`

```go
// AskQuestionStream 流式智能问答接口
func AskQuestionStream(c *gin.Context) {
    // 1. 设置 SSE 响应头
    c.Header("Content-Type", "text/event-stream")
    c.Header("Cache-Control", "no-cache")
    c.Header("Connection", "keep-alive")
    c.Header("X-Accel-Buffering", "no")

    // 2. 发送会话 ID
    c.Writer.Write([]byte("data: {\"type\":\"session\",\"session_id\":123}\n\n"))

    // 3. 逐个发送流式事件
    for event := range ch {
        jsonData, _ := json.Marshal(event)
        c.Writer.Write([]byte(fmt.Sprintf("data: %s\n\n", jsonData)))
        c.Writer.Flush()
    }

    // 4. 发送结束标记
    c.Writer.Write([]byte("data: [DONE]\n\n"))
}
```

## 4. 前端实现

### 4.1 流式 API 函数

**文件**: `src/services/qa.ts`

```typescript
// 流式事件类型
export interface StreamEvent {
  type: 'chunk' | 'done' | 'session' | 'error'
  content: string
  session_id?: number
  confidence?: number
  sources?: Array<{...}>
  related?: Array<{...}>
}

// 流式提问生成器
export async function* askQuestionStream(data: AskParams) {
  const response = await fetch('/api/v1/ask/stream', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify(data)
  })

  const reader = response.body!.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    // 解析 SSE 数据
    const text = decoder.decode(value)
    const lines = text.split('\n')
    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const event = JSON.parse(line.slice(6))
        yield event
      }
    }
  }
}
```

### 4.2 页面组件使用

**文件**: `src/views/qa/index.vue`

```typescript
const handleAskQuestion = async (question: string) => {
  // 1. 添加用户消息
  messages.value.push(userMsg)

  // 2. 添加空的 AI 消息占位
  messages.value.push(aiMsg)
  const aiMsgIndex = messages.value.length - 1

  // 3. 使用流式 API
  const stream = askQuestionStream({ question, conversation_id })

  for await (const event of stream) {
    if (event.type === 'chunk') {
      // 实时追加内容
      messages.value[aiMsgIndex].content += event.content
    } else if (event.type === 'done') {
      // 完成事件，更新来源信息
      messages.value[aiMsgIndex].sources = event.sources
    }
  }
}
```

## 5. SSE 事件格式

### 5.1 会话创建事件
```json
data: {"type":"session","session_id":123}
```

### 5.2 内容块事件
```json
data: {"type":"chunk","content":"需求"}
data: {"type":"chunk","content":"分析"}
data: {"type":"chunk","content":"是"}
```

### 5.3 完成事件
```json
data: {"type":"done","content":"","confidence":0.85,"sources":[...],"related":[...]}
```

### 5.4 错误事件
```json
data: {"type":"error","content":"服务暂时不可用"}
```

### 5.5 结束标记
```
data: [DONE]
```

## 6. 降级策略

当流式服务不可用时，系统会自动降级：

1. **流式知识图谱 RAG** → 失败时降级到
2. **流式普通 RAG** → 失败时降级到
3. **非流式问答** → 使用原有的 `Ask()` 方法
4. **本地关键词搜索** → 最终降级方案

## 8. 性能考虑

### 8.1 超时设置
- HTTP 客户端：无超时（流式连接）
- 前端：使用 AbortController 可选超时

### 8.2 并发处理
- 每个请求使用独立的 goroutine
- Channel 缓冲区大小：100

### 8.3 内存管理
- Channel 使用后及时关闭
- 避免内存泄漏

## 10. 总结

流式输出功能通过以下技术栈实现：

- **后端**: Go + Gin + Ollama Stream API
- **前端**: Vue 3 + Fetch API + Async Generator
- **协议**: Server-Sent Events (SSE)

该方案具有良好的用户体验，支持实时显示 AI 生成内容，并具备完善的降级策略确保系统稳定性。
