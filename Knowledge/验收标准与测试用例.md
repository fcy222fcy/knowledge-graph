## 一、用户认证模块 (Auth)

### AC-01: 用户注册
- **前置条件**: 未登录状态
- **操作**: POST `/api/v1/auth/register` 提交 username, email, password
- **验收标准**:
  - [ ] 成功注册返回 200，包含用户信息（id, username, email）
  - [ ] 重复用户名注册返回错误提示
  - [ ] 重复邮箱注册返回错误提示
  - [ ] 密码长度不足 6 位返回错误提示
  - [ ] 缺少必填字段返回错误提示

### AC-02: 用户登录
- **前置条件**: 已注册用户
- **操作**: POST `/api/v1/auth/login` 提交 username, password
- **验收标准**:
  - [ ] 正确凭据返回 200，包含 JWT token 和用户信息
  - [ ] 错误密码返回 401 错误
  - [ ] 不存在的用户名返回 401 错误
  - [ ] 返回的 token 可用于后续认证请求

### AC-03: Token 刷新
- **前置条件**: 已登录，持有有效 token
- **操作**: POST `/api/v1/auth/refresh` 携带 Bearer token
- **验收标准**:
  - [ ] 有效 token 刷新返回新 token
  - [ ] 过期 token 返回 401

### AC-04: 用户信息
- **前置条件**: 已登录
- **操作**: GET `/api/v1/users/profile`
- **验收标准**:
  - [ ] 返回当前用户的 profile 信息
  - [ ] 未认证请求返回 401

## 二、资料管理模块 (Documents)

### AC-05: 文件上传
- **前置条件**: 已登录
- **操作**: POST `/api/v1/documents` 上传 .md 文件
- **验收标准**:
  - [ ] 上传成功返回 200，包含文件 id、标题、状态
  - [ ] 文件存储到服务端 uploads 目录
  - [ ] 数据库记录 status = "completed"（.md/.txt 直接解析）
  - [ ] 拒绝不支持的文件类型
  - [ ] 超大文件（>10MB）返回错误

### AC-06: 文件列表
- **前置条件**: 已登录，已上传文件
- **操作**: GET `/api/v1/documents?page=1&size=10`
- **验收标准**:
  - [ ] 返回分页文件列表
  - [ ] 列表包含文件标题、类型、大小、状态、上传时间
  - [ ] 支持 keyword 搜索过滤
  - [ ] 支持 status 状态过滤

### AC-07: 文件详情
- **前置条件**: 已登录，已上传文件
- **操作**: GET `/api/v1/documents/:id`
- **验收标准**:
  - [ ] 返回文件完整信息
  - [ ] 不存在的 id 返回 404

### AC-08: 文件内容预览
- **前置条件**: 已登录，已上传 .md 文件
- **操作**: GET `/api/v1/documents/:id/content`
- **验收标准**:
  - [ ] 返回文件文本内容
  - [ ] .md 文件返回 Markdown 原文

### AC-09: 文件删除
- **前置条件**: 已登录，已上传文件
- **操作**: DELETE `/api/v1/documents/:id`
- **验收标准**:
  - [ ] 删除成功返回 200
  - [ ] 再次查询返回 404
  - [ ] 删除他人文件返回 403

## 三、知识图谱模块 (Graph)

### AC-10: 获取图谱数据
- **前置条件**: 已登录，有构建好的知识图谱
- **操作**: GET `/api/v1/graph`
- **验收标准**:
  - [ ] 返回 nodes 和 edges 数组
  - [ ] nodes 包含 id, name, type/category 字段
  - [ ] edges 包含 source, target, relation 字段
  - [ ] 无图谱数据时返回空数组（不报错）

### AC-11: 图谱搜索
- **前置条件**: 已登录，有图谱数据
- **操作**: GET `/api/v1/graph?keyword=需求`
- **验收标准**:
  - [ ] 返回匹配关键词的节点和关联边
  - [ ] 不匹配时返回空结果

### AC-12: 图谱关系筛选
- **前置条件**: 已登录，有图谱数据
- **操作**: GET `/api/v1/graph?relation_type=DEPENDS_ON`
- **验收标准**:
  - [ ] 只返回指定关系类型的边和关联节点

### AC-13: 构建知识图谱
- **前置条件**: 已登录，有已上传且解析完成的文档
- **操作**: POST `/api/v1/graph/build` 携带 document_ids
- **验收标准**:
  - [ ] 触发构建流程，返回构建任务信息
  - [ ] 构建状态可在 build history 中查询
  - [ ] 无文档 id 时返回错误提示

### AC-14: 构建历史
- **前置条件**: 已登录
- **操作**: GET `/api/v1/graph/build/history`
- **验收标准**:
  - [ ] 返回构建历史列表
  - [ ] 包含状态、创建时间、结果摘要

## 四、智能问答模块 (QA/Ask)

### AC-15: 创建会话
- **前置条件**: 已登录
- **操作**: POST `/api/v1/ask/sessions`
- **验收标准**:
  - [ ] 创建成功返回会话 id 和标题

### AC-16: 会话列表
- **前置条件**: 已登录，有历史会话
- **操作**: GET `/api/v1/ask/sessions?page=1&size=10`
- **验收标准**:
  - [ ] 返回分页会话列表
  - [ ] 包含会话标题、创建时间

### AC-17: 会话消息
- **前置条件**: 已登录，有会话及消息
- **操作**: GET `/api/v1/ask/sessions/:id/messages`
- **验收标准**:
  - [ ] 返回该会话的所有消息
  - [ ] 消息包含 role, content, created_at

### AC-18: 提问（有 AI 服务）
- **前置条件**: 已登录，Python AI 服务运行中，有知识图谱数据
- **操作**: POST `/api/v1/ask` 提交 question
- **验收标准**:
  - [ ] 返回 200，包含 answer 文本
  - [ ] 返回 source 知识来源列表（关联的文档/知识点）
  - [ ] 返回 session_id 用于关联会话
  - [ ] 回答内容与软件工程领域相关（非空、非无关内容）

### AC-19: 提问（无 AI 服务降级）
- **前置条件**: 已登录，Python AI 服务未运行，有文档和知识点数据
- **操作**: POST `/api/v1/ask` 提交 question
- **验收标准**:
  - [ ] 不报错，返回降级回答
  - [ ] 回答基于本地关键词匹配或知识点匹配

### AC-20: 历史问答
- **前置条件**: 已登录，有问答历史
- **操作**: GET `/api/v1/ask/history?page=1&size=10`
- **验收标准**:
  - [ ] 返回分页问答历史
  - [ ] 包含 question, answer, session_id, created_at
