# 知识图谱构建功能设计

## 功能目标

本功能用于完成“资料输入 -> Python AI 服务抽取 -> Neo4j 图谱存储 -> FAISS 向量存储 -> 前端图谱展示 -> 语义问答引用”的答辩闭环。

Neo4j 和 FAISS 是当前实现的核心能力：

- Neo4j 保存知识点节点和知识点之间的关系。
- FAISS 保存资料分块向量，用于语义检索。
- Go 后端作为业务 API 网关，负责读取资料、调用 Python AI 服务、记录问答数据。
- MySQL 继续保存用户、资料、题库、问答记录和学习分析数据。

## 当前实现

- `POST /api/v1/graph/build`：按资料 ID 触发构建。
- Go 后端读取资料描述和本地示例文档内容。
- Go 后端调用 Python AI 服务 `/build`。
- Python AI 服务抽取知识点、关系和文本分块。
- Python AI 服务将节点和关系写入 Neo4j。
- Python AI 服务将分块向量写入 FAISS。
- Go 后端同步一份节点和关系到 MySQL，兼容现有统计和旧接口。
- `GET /api/v1/graph`：优先从 Python AI 服务读取 Neo4j 图谱。
- `POST /api/v1/ask`：优先通过 Python AI 服务调用 FAISS 语义检索，再返回引用和相关知识点。

## Python AI 服务接口

构建图谱：

```http
POST http://127.0.0.1:5000/build
Content-Type: application/json

{
  "document_id": 1,
  "title": "软件工程课堂笔记",
  "content": "需求分析用于识别和确认用户需求。软件测试用于验证系统是否满足需求。",
  "source": "课堂资料"
}
```

响应：

```json
{
  "document_id": 1,
  "document_title": "软件工程课堂笔记",
  "created_points": 2,
  "created_relations": 1,
  "chunk_count": 1,
  "vector_count": 1,
  "status": "completed",
  "message": "知识图谱构建完成",
  "nodes": [],
  "edges": []
}
```

语义检索：

```http
POST http://127.0.0.1:5000/search
Content-Type: application/json

{
  "query": "如何确认需求是否满足",
  "top_k": 3
}
```

图谱读取：

```http
GET http://127.0.0.1:5000/graph
```

健康检查：

```http
GET http://127.0.0.1:5000/health
```

## Go 后端接口

前端仍调用原有路径：

```http
POST /api/v1/graph/build
GET /api/v1/graph
POST /api/v1/ask
```

`/api/v1/graph/build` 响应增加：

```json
{
  "chunk_count": 3,
  "vector_count": 12
}
```

前端在构建结果中展示“分块”和“向量”指标。

## 抽取与检索策略

本地答辩环境不依赖在线大模型：

- 文本抽取采用确定性课程术语规则。
- 文本分块按长度和句子边界切分。
- 向量化采用本地 Hashing 向量化。
- FAISS 使用 `IndexFlatIP` 对归一化向量做相似度检索。
- 命中文本分块后，根据分块关联的知识点在 Neo4j 中扩展相关节点。

该方案不是样例假数据：Neo4j 和 FAISS 都会真实写入、查询和持久化。后续可把 Hashing 向量化替换为更强的 embedding 模型，接口不需要大改。

## 验收标准

- 空资料构建时返回明确错误。
- 有效资料构建后，Neo4j 中出现 `KnowledgePoint` 节点和 `RELATED` 关系。
- 有效资料构建后，FAISS `vector_count` 大于 0。
- 前端知识图谱页面展示点和线。
- 提问语义相关但关键词不完全一致的问题时，系统能返回引用来源。
- 问答记录仍写入 MySQL，学习分析页面能统计提问次数。
- `python -m pytest python-ai-service/tests -v` 通过。
- `go test ./...` 通过。
- `npm run build` 通过。
