# 前端开发跟进

本文档用于记录前端开发计划、每次执行的命令、修改的文件、验证结果和后续待办。由于项目不可能一次性完成，后续每一轮前端开发都应先更新本文件，再进行代码修改。

## 1. 总体目标

先完成一个可运行、可演示、后续可对接后端的 Vue 前端。

前端第一阶段不追求所有功能完全真实联通，优先实现：

- 清晰的应用主界面
- 核心模块入口
- 问答、资源、图谱等关键页面的演示闭环
- 后续可替换 mock 数据为真实 API 的代码结构

## 2. 阶段计划

### 阶段一：前端工程基础

目标：让前端项目可以稳定启动、构建，并形成基本目录规范。

计划内容：

- 初始化 Vue 3 + Vite 工程
- 整理 `src` 目录结构
- 建立全局样式入口
- 建立基础应用布局
- 预留 API 请求封装位置
- 验证 `npm install`、`npm run dev`、`npm run build`

交付结果：

- 可启动的前端项目
- 基础页面骨架
- 明确的文件组织方式

### 阶段二：主界面与路由结构

目标：搭建平台的主要信息架构。

计划页面：

- 知识问答
- 知识图谱
- 资源管理
- 题库练习
- 学习分析
- 管理控制台

计划内容：

- 左侧或顶部导航
- 页面标题和当前模块状态
- 统一空状态、加载状态、错误状态
- 移动端基础适配

### 阶段三：知识问答核心页

目标：优先做出平台最核心的使用体验。

计划内容：

- 问题输入框
- 提问按钮和快捷问题
- 答案展示区域
- 引用来源片段
- 置信度或相关度展示
- 相关知识点列表
- mock 问答数据

### 阶段四：资源管理页

目标：展示课程资料上传与处理流程。

计划内容：

- 文件选择与上传入口
- 文档列表
- 处理状态：待处理、处理中、完成、失败
- 文档元信息展示
- 后续对接 `/api/v1/documents`

### 阶段五：知识图谱页

目标：先完成轻量级图谱展示，后续再替换为真实图谱可视化库。

计划内容：

- 节点和关系列表
- 核心知识点关系图的静态或轻交互展示
- 节点详情面板
- 后续对接 `/api/v1/graph/nodes`、`/api/v1/graph/edges`

### 阶段六：题库与学习分析

目标：补齐教学闭环。

计划内容：

- 题目列表
- 答题交互
- 答案反馈
- 错题状态
- 学习统计卡片
- 知识点掌握度展示

### 阶段七：后端 API 对接

目标：将 mock 数据逐步替换成真实接口。

计划内容：

- 统一 API 客户端
- 请求错误处理
- 登录态处理
- 环境变量配置
- 各页面分批接入真实接口

## 3. 当前前端建议技术栈

- Vue 3
- Vite
- 原生 CSS 或后续引入组件库
- 后续可按需要加入 Vue Router、Pinia、Axios

当前建议：先不要过早引入太多库，等页面结构稳定后再补充。

## 6. 下一步建议

下一次正式开始写前端时，建议从“阶段一：前端工程基础”继续，先处理以下事情：

- 确认是否保留当前 Vite 初始化结果
- 清理默认示例页面
- 设计基础目录结构
- 建立主布局和导航
- 执行一次构建验证
