# 计算机组成原理详解

## 一、计算机系统概述

### 1.1 冯·诺依曼体系结构

```
┌─────────────────────────────────────────────────┐
│                   输入设备                        │
│              (键盘、鼠标、摄像头)                  │
└─────────────────────┬───────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────┐
│                      CPU                         │
│  ┌───────────────┐    ┌─────────────────────┐   │
│  │   控制器(CU)   │    │    运算器(ALU)       │   │
│  │  - 程序计数器PC │    │  - 算术运算          │   │
│  │  - 指令寄存器IR│    │  - 逻辑运算          │   │
│  │  - 地址寄存器  │    │  - 移位运算          │   │
│  └───────────────┘    └─────────────────────┘   │
│  ┌───────────────────────────────────────────┐  │
│  │              寄存器组                      │  │
│  │  ACC(累加器) | PC(程序计数器) | IR(指令寄存器)│  │
│  └───────────────────────────────────────────┘  │
└─────────────────────┬───────────────────────────┘
                      │ 系统总线
          ┌───────────┼───────────┐
          │           │           │
          ▼           ▼           ▼
┌──────────────┐ ┌─────────┐ ┌──────────────┐
│   主存储器    │ │  输出设备 │ │  辅助存储器   │
│  (RAM/ROM)   │ │(显示器、 │ │ (硬盘、SSD)  │
│              │ │ 打印机)  │ │              │
└──────────────┘ └─────────┘ └──────────────┘
```

### 1.2 冯·诺依曼计算机特点

1. **存储程序**：指令和数据以同等地位存储在存储器中
2. **二进制**：指令和数据都用二进制表示
3. **五大部件**：运算器、控制器、存储器、输入设备、输出设备
4. **指令由操作码和地址码组成**
5. **以运算器为中心**（现代计算机以存储器为中心）

### 1.3 计算机性能指标

| 指标 | 公式/说明 | 单位 |
|------|----------|------|
| 时钟周期 | T = 1/f | 秒(s) |
| 主频 | f = 1/T | Hz, GHz |
| CPI | Clock Cycles Per Instruction | - |
| MIPS | Million Instructions Per Second | M/s |
| MFLOPS | Million FLOating-point Per Second | M/s |
| CPU执行时间 | T = IC × CPI × T_clk | 秒(s) |
| 吞吐量 | 单位时间内完成的任务数 | 任务/s |

## 二、数据表示

### 2.1 进制转换

```
十进制 → 二进制: 除2取余法
例: 13 → 1101
13/2 = 6 ... 1
 6/2 = 3 ... 0
 3/2 = 1 ... 1
 1/2 = 0 ... 1
结果: 1101

二进制 → 十六进制: 每4位一组
例: 1101 0011 = D3
    1101 = D (13)
    0011 = 3

二进制 → 八进制: 每3位一组
例: 110 100 110 = 646
```

### 2.2 原码、反码、补码

以8位为例，表示 -5：

| 编码方式 | 正数 | 负数 |
|---------|------|------|
| 原码 | 0000 0101 | 1000 0101 |
| 反码 | 0000 0101 | 1111 1010 |
| 补码 | 0000 0101 | 1111 1011 |

**补码的优势**：
1. 0 的表示唯一（原码中 +0 和 -0 不同）
2. 加减法统一运算
3. 符号位参与运算

**补码范围**（n 位）：
- 正数范围：0 到 2^(n-1) - 1
- 负数范围：-2^(n-1) 到 -1
- 8位：-128 到 127
- 16位：-32768 到 32767

### 2.3 浮点数表示

IEEE 754 标准：

```
单精度（32位）:
┌─────┬──────────┬──────────────────┐
│ 符号 │ 指数(8位) │     尾数(23位)    │
│  1位 │  偏移127  │     隐含前导1     │
└─────┴──────────┴──────────────────┘

双精度（64位）:
┌─────┬───────────┬──────────────────────────────┐
│ 符号 │ 指数(11位) │         尾数(52位)           │
│  1位 │  偏移1023  │         隐含前导1             │
└─────┴───────────┴──────────────────────────────┘

数值 = (-1)^S × 1.M × 2^(E-偏移量)

例: 单精度表示 -6.5
符号: 1 (负数)
6.5 = 110.1 = 1.101 × 2^2
指数: 2 + 127 = 129 = 10000001
尾数: 101000...000 (23位)

结果: 1 10000001 10100000000000000000000
     = C0A00000 (十六进制)
```

### 2.4 溢出检测

```
两个正数相加得负数 → 正溢出
两个负数相加得正数 → 负溢出

例: 8位补码
  0111 1111 (127)
+ 0000 0001 (1)
= 1000 0000 (-128) ← 溢出!

检测方法:
符号位进位 ⊕ 最高位进位 = 1 → 溢出
```

## 三、指令系统

### 3.1 指令格式

```
┌─────────────────┬──────────────────┐
│    操作码(Opcode) │    地址码         │
│    指明做什么操作  │    操作数地址      │
└─────────────────┴──────────────────┘

按地址码数量分类:
三地址: OP A1, A2, A3    → A3 = A1 OP A2
二地址: OP A1, A2        → A1 = A1 OP A2
一地址: OP A1            → ACC = ACC OP A1
零地址: OP               → 栈操作
```

### 3.2 寻址方式

| 寻址方式 | 有效地址 | 特点 |
|---------|---------|------|
| 立即寻址 | 操作数在指令中 | 速度快，灵活性差 |
| 直接寻址 | EA = A | 简单，地址空间有限 |
| 间接寻址 | EA = (A) | 灵活，需两次访问内存 |
| 寄存器寻址 | EA = R | 速度快，寄存器数量有限 |
| 寄存器间接寻址 | EA = (R) | 灵活，一次内存访问 |
| 基址寻址 | EA = (BR) + A | 程序重定位 |
| 变址寻址 | EA = (IX) + A | 数组访问 |
| 相对寻址 | EA = (PC) + A | 程序计数器相对 |

**变址寻址访问数组**：
```
基地址: 数组起始地址 = 1000
变址寄存器 IX: 数组下标 i

EA = 1000 + i

arr[0] → EA = 1000 + 0 = 1000
arr[1] → EA = 1000 + 1 = 1001
arr[2] → EA = 1000 + 2 = 1002
```

### 3.3 CISC vs RISC

| 特性 | CISC | RISC |
|------|------|------|
| 指令数量 | 多（数百条） | 少（几十条） |
| 指令长度 | 可变 | 固定 |
| 指令格式 | 多种 | 少数几种 |
| 寻址方式 | 复杂多样 | 简单 |
| 访存方式 | 可直接访存 | Load/Store |
| 流水线 | 不易实现 | 容易实现 |
| 编译器 | 简单 | 复杂 |
| 功耗 | 较高 | 较低 |
| 代表 | x86, x86-64 | ARM, MIPS, RISC-V, LoongArch |

## 四、CPU 组织与功能

### 4.1 CPU 数据通路

```
┌──────────────────────────────────────────────────┐
│                     CPU                           │
│                                                   │
│  ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐        │
│  │ PC  │──→│ MAR │──→│内存  │──→│ MDR │        │
│  └─────┘   └─────┘   └─────┘   └──┬──┘        │
│     │                              │             │
│     │          ┌─────┐             │             │
│     └─────────→│ IR  │←────────────┘             │
│                └──┬──┘                            │
│                   │                               │
│            ┌──────▼──────┐                        │
│            │   控制器 CU  │                        │
│            └──────┬──────┘                        │
│                   │                               │
│    ┌──────────────┼──────────────┐                │
│    │              │              │                │
│    ▼              ▼              ▼                │
│ ┌─────┐      ┌─────┐      ┌─────┐               │
│ │  ALU │      │ ACC │      │ 通用 │               │
│ │ 运算器│      │累加器│      │寄存器│               │
│ └─────┘      └─────┘      └─────┘               │
└──────────────────────────────────────────────────┘
```

### 4.2 指令执行过程

```
取指周期（Fetch）:
  PC → MAR → 地址总线 → 存储器
  存储器 → 数据总线 → MDR → IR
  PC ← PC + 1

间址周期（Indirect）:
  IR(地址码) → MAR → 存储器
  存储器 → MDR → IR(地址码)

执行周期（Execute）:
  根据 IR 中的操作码执行相应操作
  如: 加法 → ALU 执行加法，结果存入 ACC

中断周期（Interrupt）:
  检查是否有中断请求
  有 → 保存断点，转中断服务程序
  无 → 返回取指周期
```

### 4.3 流水线技术

**四级流水线**：
```
指令:   I1    I2    I3    I4    I5    I6

时钟1:  取指
时钟2:  译码  取指
时钟3:  执行  译码  取指
时钟4:  写回  执行  译码  取指
时钟5:        写回  执行  译码  取指
时钟6:              写回  执行  译码  取指
时钟7:                    写回  执行  译码
时钟8:                          写回  执行
时钟9:                                写回

无流水线: 6 × 4 = 24 个时钟周期
有流水线: 4 + 5 = 9 个时钟周期
加速比: 24 / 9 ≈ 2.67
```

**流水线冲突**：

| 冲突类型 | 原因 | 解决方法 |
|---------|------|---------|
| 结构冲突 | 多指令争用同一硬件 | 资源复制、流水线停顿 |
| 数据冲突 | 指令间存在数据依赖 | 数据转发、流水线停顿、编译器调度 |
| 控制冲突 | 分支指令改变程序流 | 分支预测、延迟槽、提前计算 |

**分支预测**：
```
静态预测:
- 总是预测不跳转（Not Taken）
- 总是预测跳转（Taken）

动态预测:
- 1位预测器: 根据上次结果
- 2位预测器: 2-bit饱和计数器
  00: 强不跳转 → 01: 弱不跳转 → 10: 弱跳转 → 11: 强跳转
```

## 五、存储系统

### 5.1 存储层次

```
┌──────────────┐  ← 寄存器（0.3ns，几百字节）
├──────────────┤
│   L1 Cache   │  ← 1-2ns，32-64KB
├──────────────┤
│   L2 Cache   │  ← 3-10ns，256KB-1MB
├──────────────┤
│   L3 Cache   │  ← 10-20ns，几MB
├──────────────┤
│    主存       │  ← 50-100ns，几GB-几十GB
├──────────────┤
│  SSD/硬盘     │  ← 100μs-10ms，几百GB-几TB
├──────────────┤
│  磁带/光盘    │  ← 秒级，TB级
└──────────────┘

设计原理: 局部性原理
- 时间局部性: 近期访问的数据可能再次被访问
- 空间局部性: 临近地址的数据可能被访问
```

### 5.2 Cache 工作原理

**Cache 地址映射**：

| 映射方式 | 原理 | 优点 | 缺点 |
|---------|------|------|------|
| 直接映射 | 主存块只能放到固定 Cache 行 | 简单，硬件成本低 | 冲突率高 |
| 全相联映射 | 主存块可放到任意 Cache 行 | 冲突率低 | 硬件成本高 |
| 组相联映射 | 主存块放到指定组中的任意行 | 折中方案 | 硬件成本适中 |

**Cache 替换算法**：

```
直接映射: 无需替换（直接替换）

全相联/组相联:
- LRU (最近最少使用): 性能好，实现复杂
- FIFO (先进先出): 简单，可能出现 Belady 异常
- 随机替换: 简单，性能尚可
```

**Cache 写策略**：

| 策略 | 写命中 | 写不命中 | 特点 |
|------|-------|---------|------|
| 写直达 | 同时写 Cache 和主存 | 需要调块 | 简单，总线流量大 |
| 写回 | 只写 Cache，脏位标记 | 需要调块 | 复杂，总线流量小 |

### 5.3 Cache 命中率计算

```
设 Cache 命中率为 h，Cache 访问时间为 t_c，主存访问时间为 t_m

平均访问时间 = h × t_c + (1-h) × t_m

例: h = 0.95, t_c = 1ns, t_m = 100ns
平均访问时间 = 0.95 × 1 + 0.05 × 100 = 5.95ns

Cache-主存 系统的加速比 = t_m / t_avg = 100 / 5.95 ≈ 16.8
```

### 5.4 虚拟存储器

**地址转换**：
```
CPU 发出的地址 → 虚拟地址（逻辑地址）
                    │
                    ▼
              ┌──────────┐
              │  TLB     │ ← 快表（页表缓存）
              └────┬─────┘
                   │ 命中
                   ▼
              物理地址 → 访问主存

              TLB 未命中 → 查页表 → 物理地址
                              │
                              ▼
                        更新 TLB
```

**多级页表**：
```
32位系统，4KB页大小:
虚拟地址: [一级页号(10位)] [二级页号(10位)] [页内偏移(12位)]

一级页表: 1024个表项，每项指向一个二级页表
二级页表: 1024个表项，每项指向一个物理帧

优点: 只有被使用的二级页表才占用内存
```

## 七、输入输出系统

### 7.1 I/O 控制方式

| 方式 | 数据传输 | CPU 参与 | 特点 |
|------|---------|---------|------|
| 程序查询 | 字/字节 | 完全参与 | 简单，CPU 利用率低 |
| 中断方式 | 字/字节 | 每次传输中断一次 | 比查询好，中断开销大 |
| DMA | 块 | 仅初始化和结束 | 传输数据不经过 CPU |
| 通道 | 块/组 | 极少 | 专用处理器，效率最高 |

### 7.2 DMA 工作原理

```
1. CPU 向 DMA 控制器设置参数:
   - 主存起始地址
   - 传输数据量
   - 设备地址
   - 传输方向

2. CPU 启动 DMA 传输

3. DMA 控制器接管总线，直接与主存和设备交互
   设备 → DMA控制器 → 主存 (或 主存 → DMA控制器 → 设备)

4. 传输完成，DMA 控制器向 CPU 发中断

5. CPU 响应中断，处理传输结果
```

**DMA vs 中断方式对比**：

| 对比项 | 中断方式 | DMA 方式 |
|--------|---------|---------|
| 数据传输 | CPU 搬运 | DMA 控制器搬运 |
| 每个数据中断 | 是 | 否（整块） |
| CPU 占用 | 高 | 低 |
| 适用场景 | 低速设备 | 高速设备（磁盘、网卡） |

### 7.3 I/O 接口

```
CPU ←→ I/O 接口 ←→ I/O 设备

I/O 接口功能:
1. 设备选择: 通过设备地址识别
2. 数据缓冲: 匹配速度差异
3. 信号转换: 电平、串并转换
4. 控制逻辑: 协调工作时序
5. 状态报告: 忙、就绪、错误

端口编址方式:
- 统一编址: I/O 端口和内存统一编址（ARM）
- 独立编址: I/O 端口和内存分开编址（x86）
```

## 八、中断系统

### 8.1 中断分类

```
中断
├── 内中断（异常/Exception）
│   ├── 故障（Fault）: 可修复，如缺页异常
│   ├── 陷阱（Trap）: 有意触发，如系统调用
│   └── 终止（Abort）: 不可修复，如硬件错误
│
└── 外中断（Interrupt）
    ├── 可屏蔽中断（INTR）: 可被屏蔽，如 I/O 中断
    └── 不可屏蔽中断（NMI）: 不可屏蔽，如电源故障
```

### 8.2 中断响应过程

```
1. 中断请求
   外设或部件发出中断信号

2. 中断判优
   多个中断同时发生时，按优先级排队
   高优先级中断可以打断低优先级中断服务

3. 中断响应（硬件自动完成，中断隐指令）
   a. 关中断 (EINT ← 0)
   b. 保存断点 (PC、PSW 压栈)
   c. 获取中断向量地址
   d. 转入中断服务程序

4. 中断服务程序（ISR）
   保存现场 → 处理中断 → 恢复现场

5. 中断返回
   恢复断点，开中断，返回原程序
```

### 8.3 中断嵌套

```
主程序
  │
  ▼
中断A（低优先级）
  │
  ▼
中断B（高优先级）← 嵌套
  │
  ▼
返回中断A
  │
  ▼
返回主程序
```

## 十、计算机组成原理面试题

### Q1: 冯·诺依曼计算机的核心思想是什么？

**答**：存储程序。指令和数据以同等地位存储在存储器中，计算机按照程序顺序执行指令。

### Q2: 什么是 Cache？为什么需要 Cache？

**答**：
- Cache 是高速缓冲存储器，位于 CPU 和主存之间
- 目的：解决 CPU 和主存速度不匹配的问题
- 原理：利用程序的局部性原理（时间局部性和空间局部性）
- 效果：Cache 命中率通常在 95% 以上，显著提高平均访问速度

### Q3: 什么是流水线？流水线冲突有哪些？

**答**：
流水线是将指令执行过程分成多个阶段，让多条指令重叠执行的技术。

三种冲突：
1. **结构冲突**：多条指令争用同一硬件资源（如同时访存）
2. **数据冲突**：后执行的指令需要用到前面指令的结果
3. **控制冲突**：分支指令改变程序执行流

### Q4: DMA 和中断方式的区别？

**答**：
- **中断方式**：每传输一个数据就要中断一次 CPU，CPU 参与数据搬运
- **DMA 方式**：DMA 控制器直接与主存交互传输数据块，CPU 只负责初始化和结束处理
- DMA 适用于高速设备（如磁盘、网卡），中断方式适用于低速设备

### Q5: 什么是虚拟内存？

**答**：
虚拟内存为每个进程提供独立的、连续的逻辑地址空间，逻辑地址空间可以大于物理内存。通过页表将逻辑地址映射到物理地址，使用按需分页和页面置换算法，使得有限的物理内存能够支持更大的程序运行。

### Q6: CISC 和 RISC 的区别？

**答**：
- CISC（复杂指令集）：指令数量多、长度可变、寻址方式复杂、支持直接访存
- RISC（精简指令集）：指令数量少、长度固定、寻址方式简单、仅 Load/Store 访存
- RISC 更适合流水线实现，功耗更低（ARM 广泛用于移动设备）
- RISC-V 是开源的 RISC 指令集架构，近年发展迅速
