```
├── 结构型模式（7个）── 如何组合对象？
```

## 二、创建型模式

### 2.1 单例模式（Singleton）

**意图**：确保一个类只有一个实例，并提供全局访问点。

**Go 实现**（利用 sync.Once）：
```go
package singleton

import "sync"

type Database struct {
    host string
    port int
}

var (
    instance *Database
    once     sync.Once
)

func GetInstance() *Database {
    once.Do(func() {
        instance = &Database{
            host: "localhost",
            port: 3306,
        }
    })
    return instance
}
```

**应用场景**：
- 数据库连接池
- 日志记录器
- 配置管理器
- 线程池

**注意事项**：
- 饿汉式（程序启动时创建）：线程安全，可能浪费资源
- 懒汉式（首次使用时创建）：需要加锁保证线程安全
- 双重检查锁（DCL）：兼顾懒加载和线程安全

### 2.2 工厂方法模式（Factory Method）

**意图**：定义创建对象的接口，让子类决定实例化哪个类。

```go
// 产品接口
type Transport interface {
    Deliver() string
}

// 具体产品
type Truck struct{}
func (t *Truck) Deliver() string { return "公路运输货物" }

type Ship struct{}
func (s *Ship) Deliver() string { return "海运货物" }

type Airplane struct{}
func (a *Airplane) Deliver() string { return "航空运输货物" }

// 工厂接口
type Logistics interface {
    CreateTransport() Transport
    PlanDelivery() string
}

// 具体工厂
type RoadLogistics struct{}
func (r *RoadLogistics) CreateTransport() Transport { return &Truck{} }
func (r *RoadLogistics) PlanDelivery() string {
    return "计划: " + r.CreateTransport().Deliver()
}

type SeaLogistics struct{}
func (s *SeaLogistics) CreateTransport() Transport { return &Ship{} }
func (s *SeaLogistics) PlanDelivery() string {
    return "计划: " + s.CreateTransport().Deliver()
}

// 使用
func main() {
    var logistics Logistics
    logistics = &RoadLogistics{}
    fmt.Println(logistics.PlanDelivery()) // 计划: 公路运输货物

    logistics = &SeaLogistics{}
    fmt.Println(logistics.PlanDelivery()) // 计划: 海运货物
}
```

### 2.3 抽象工厂模式（Abstract Factory）

**意图**：提供创建一系列相关对象的接口，无需指定具体类。

```go
// 抽象产品
type Button interface {
    Render() string
}

type Checkbox interface {
    Render() string
}

// 具体产品 - Windows 风格
type WindowsButton struct{}
func (b *WindowsButton) Render() string { return "[Windows Button]" }

type WindowsCheckbox struct{}
func (c *WindowsCheckbox) Render() string { return "[Windows Checkbox]" }

// 具体产品 - macOS 风格
type MacButton struct{}
func (b *MacButton) Render() string { return "(MacOS Button)" }

type MacCheckbox struct{}
func (c *MacCheckbox) Render() string { return "(MacOS Checkbox)" }

// 抽象工厂
type GUIFactory interface {
    CreateButton() Button
    CreateCheckbox() Checkbox
}

// 具体工厂
type WindowsFactory struct{}
func (f *WindowsFactory) CreateButton() Button     { return &WindowsButton{} }
func (f *WindowsFactory) CreateCheckbox() Checkbox { return &WindowsCheckbox{} }

type MacFactory struct{}
func (f *MacFactory) CreateButton() Button     { return &MacButton{} }
func (f *MacFactory) CreateCheckbox() Checkbox { return &MacCheckbox{} }
```

### 2.4 建造者模式（Builder）

**意图**：将复杂对象的构建与表示分离，允许分步骤创建。

```go
type Server struct {
    Host     string
    Port     int
    MaxConn  int
    Timeout  time.Duration
    SSL      bool
    LogPath  string
}

type ServerBuilder struct {
    server Server
}

func NewServerBuilder() *ServerBuilder {
    return &ServerBuilder{}
}

func (b *ServerBuilder) Host(host string) *ServerBuilder {
    b.server.Host = host
    return b
}

func (b *ServerBuilder) Port(port int) *ServerBuilder {
    b.server.Port = port
    return b
}

func (b *ServerBuilder) MaxConn(max int) *ServerBuilder {
    b.server.MaxConn = max
    return b
}

func (b *ServerBuilder) Timeout(t time.Duration) *ServerBuilder {
    b.server.Timeout = t
    return b
}

func (b *ServerBuilder) EnableSSL() *ServerBuilder {
    b.server.SSL = true
    return b
}

func (b *ServerBuilder) Build() Server {
    // 设置默认值
    if b.server.Port == 0 {
        b.server.Port = 8080
    }
    if b.server.MaxConn == 0 {
        b.server.MaxConn = 100
    }
    return b.server
}

// 使用
server := NewServerBuilder().
    Host("localhost").
    Port(3000).
    MaxConn(200).
    Timeout(30 * time.Second).
    EnableSSL().
    Build()
```

### 2.5 原型模式（Prototype）

**意图**：通过复制已有对象来创建新对象。

```go
type Cloneable interface {
    Clone() Cloneable
}

type Address struct {
    City    string
    Street  string
}

type Employee struct {
    Name    string
    Age     int
    Address *Address
}

func (e *Employee) Clone() Cloneable {
    // 深拷贝
    addrCopy := *e.Address
    return &Employee{
        Name:    e.Name,
        Age:     e.Age,
        Address: &addrCopy,
    }
}
```

## 三、结构型模式

### 3.1 适配器模式（Adapter）

**意图**：将一个类的接口转换成客户端期望的另一个接口。

```go
// 第三方支付接口
type ThirdPartyPayment struct{}

func (p *ThirdPartyPayment) MakeAlipayPayment(amount float64) string {
    return fmt.Sprintf("支付宝支付: %.2f元", amount)
}

func (p *ThirdPartyPayment) MakeWechatPayment(amount float64) string {
    return fmt.Sprintf("微信支付: %.2f元", amount)
}

// 目标接口
type PaymentProcessor interface {
    Pay(amount float64) string
}

// 适配器
type AlipayAdapter struct {
    thirdParty *ThirdPartyPayment
}

func (a *AlipayAdapter) Pay(amount float64) string {
    return a.thirdParty.MakeAlipayPayment(amount)
}

type WechatAdapter struct {
    thirdParty *ThirdPartyPayment
}

func (a *WechatAdapter) Pay(amount float64) string {
    return a.thirdParty.MakeWechatPayment(amount)
}
```

### 3.2 装饰器模式（Decorator）

**意图**：动态地为对象添加职责，比继承更灵活。

```go
// 基础接口
type DataSource interface {
    WriteData(data string)
    ReadData() string
}

// 基础实现
type FileDataSource struct {
    filename string
}

func (f *FileDataSource) WriteData(data string) {
    fmt.Printf("写入文件 %s: %s\n", f.filename, data)
}

func (f *FileDataSource) ReadData() string {
    return fmt.Sprintf("从文件 %s 读取的数据", f.filename)
}

// 装饰器基类
type DataSourceDecorator struct {
    wrapped DataSource
}

// 加密装饰器
type EncryptionDecorator struct {
    DataSourceDecorator
}

func (e *EncryptionDecorator) WriteData(data string) {
    encrypted := "ENC(" + data + ")"
    fmt.Println("加密数据:", encrypted)
    e.wrapped.WriteData(encrypted)
}

func (e *EncryptionDecorator) ReadData() string {
    data := e.wrapped.ReadData()
    return strings.TrimPrefix(strings.TrimSuffix(data, ")"), "ENC(")
}

// 压缩装饰器
type CompressionDecorator struct {
    DataSourceDecorator
}

func (c *CompressionDecorator) WriteData(data string) {
    compressed := "ZIP(" + data + ")"
    fmt.Println("压缩数据:", compressed)
    c.wrapped.WriteData(compressed)
}

func (c *CompressionDecorator) ReadData() string {
    data := c.wrapped.ReadData()
    return strings.TrimPrefix(strings.TrimSuffix(data, ")"), "ZIP(")
}

// 使用：链式装饰
source := &FileDataSource{"data.txt"}
// 加密 + 压缩
decorated := &CompressionDecorator{
    DataSourceDecorator{&EncryptionDecorator{
        DataSourceDecorator{source},
    }},
}
decorated.WriteData("敏感数据")
// 输出:
// 压缩数据: ZIP(ENC(敏感数据))
// 加密数据: ENC(敏感数据)
// 写入文件 data.txt: ZIP(ENC(敏感数据))
```

### 3.3 代理模式（Proxy）

**意图**：为对象提供替身/占位符，控制对该对象的访问。

```go
// 接口
type ImageLoader interface {
    Display(name string)
}

// 真实对象
type RealImage struct {
    filename string
}

func (r *RealImage) Display(name string) {
    fmt.Printf("显示图片: %s\n", name)
}

func (r *RealImage) loadFromDisk(name string) {
    fmt.Printf("从磁盘加载图片: %s\n", name)
}

// 代理
type ProxyImage struct {
    realImage *RealImage
    loaded    bool
}

func (p *ProxyImage) Display(name string) {
    if !p.loaded {
        p.realImage = &RealImage{}
        p.realImage.loadFromDisk(name)
        p.loaded = true
    }
    p.realImage.Display(name)
}
```

**代理类型**：
| 类型 | 说明 |
|------|------|
| 远程代理 | 为远程对象提供本地代表（RPC） |
| 虚拟代理 | 延迟创建开销大的对象（懒加载） |
| 保护代理 | 控制对象的访问权限 |
| 缓存代理 | 为结果提供缓存 |

### 3.4 外观模式（Facade）

**意图**：为子系统提供统一的简化接口。

```go
// 子系统
type CPU struct{}
func (c *CPU) Freeze()      { fmt.Println("CPU: 冻结") }
func (c *CPU) Jump(addr)    { fmt.Println("CPU: 跳转到", addr) }
func (c *CPU) Execute()     { fmt.Println("CPU: 执行指令") }

type Memory struct{}
func (m *Memory) Load(addr, data string) { fmt.Println("内存: 加载数据") }

type HardDrive struct{}
func (h *HardDrive) Readsector(sector, size int) string { return "数据" }

// 外观
type ComputerFacade struct {
    cpu    *CPU
    memory *Memory
    hdd    *HardDrive
}

func (f *ComputerFacade) Start() {
    fmt.Println("=== 启动计算机 ===")
    f.cpu.Freeze()
    f.memory.Load("0x00", f.hdd.Readsector(0, 1024))
    f.cpu.Jump("0x00")
    f.cpu.Execute()
    fmt.Println("=== 计算机启动完成 ===")
}
```

### 3.5 桥接模式（Bridge）

**意图**：将抽象与实现分离，使它们可以独立变化。

```go
// 实现接口
type Color interface {
    Fill() string
}

type Red struct{}
func (r *Red) Fill() string { return "红色" }

type Blue struct{}
func (b *Blue) Fill() string { return "蓝色" }

// 抽象
type Shape interface {
    Draw()
}

type Circle struct {
    color Color
}

func (c *Circle) Draw() {
    fmt.Printf("画一个 %s 的圆\n", c.color.Fill())
}

type Square struct {
    color Color
}

func (s *Square) Draw() {
    fmt.Printf("画一个 %s 的正方形\n", s.color.Fill())
}

// 使用
redCircle := &Circle{color: &Red{}}
redCircle.Draw() // 画一个 红色 的圆

blueSquare := &Square{color: &Blue{}}
blueSquare.Draw() // 画一个 蓝色 的正方形
```

### 3.6 组合模式（Composite）

**意图**：将对象组合成树形结构，统一处理单个对象和组合对象。

```go
// 组件接口
type Component interface {
    Execute() string
    GetName() string
}

// 叶子节点
type File struct {
    name string
}

func (f *File) Execute() string { return f.name }
func (f *File) GetName() string { return f.name }

// 组合节点
type Folder struct {
    name      string
    children  []Component
}

func (fo *Folder) Add(child Component) {
    fo.children = append(fo.children, child)
}

func (fo *Folder) Execute() string {
    result := fo.name + ": ["
    for i, child := range fo.children {
        if i > 0 {
            result += ", "
        }
        result += child.GetName()
    }
    result += "]"
    return result
}

func (fo *Folder) GetName() string { return fo.name }
```

## 四、行为型模式

### 4.1 策略模式（Strategy）

**意图**：定义算法族，使它们可以互相替换。

```go
// 策略接口
type SortStrategy interface {
    Sort(data []int) []int
}

// 具体策略
type BubbleSort struct{}
func (b *BubbleSort) Sort(data []int) []int {
    result := make([]int, len(data))
    copy(result, data)
    for i := 0; i < len(result); i++ {
        for j := 0; j < len(result)-i-1; j++ {
            if result[j] > result[j+1] {
                result[j], result[j+1] = result[j+1], result[j]
            }
        }
    }
    return result
}

type QuickSort struct{}
func (q *QuickSort) Sort(data []int) []int {
    result := make([]int, len(data))
    copy(result, data)
    sort.Ints(result)
    return result
}

// 上下文
type Sorter struct {
    strategy SortStrategy
}

func (s *Sorter) SetStrategy(strategy SortStrategy) {
    s.strategy = strategy
}

func (s *Sorter) Sort(data []int) []int {
    return s.strategy.Sort(data)
}

// 使用
sorter := &Sorter{}
sorter.SetStrategy(&BubbleSort{})
result := sorter.Sort([]int{3, 1, 4, 1, 5, 9})

sorter.SetStrategy(&QuickSort{})
result = sorter.Sort([]int{3, 1, 4, 1, 5, 9})
```

### 4.2 观察者模式（Observer）

**意图**：定义一对多的依赖关系，对象状态变化时自动通知。

```go
// 事件系统
type Event struct {
    Name    string
    Payload interface{}
}

type EventHandler func(Event)

type EventEmitter struct {
    listeners map[string][]EventHandler
}

func NewEventEmitter() *EventEmitter {
    return &EventEmitter{listeners: make(map[string][]EventHandler)}
}

func (e *EventEmitter) On(event string, handler EventHandler) {
    e.listeners[event] = append(e.listeners[event], handler)
}

func (e *EventEmitter) Emit(event string, payload interface{}) {
    evt := Event{Name: event, Payload: payload}
    for _, handler := range e.listeners[event] {
        handler(evt)
    }
}

// 使用
emitter := NewEventEmitter()

emitter.On("user:login", func(e Event) {
    fmt.Printf("发送欢迎邮件给用户: %v\n", e.Payload)
})

emitter.On("user:login", func(e Event) {
    fmt.Printf("记录登录日志: %v\n", e.Payload)
})

emitter.Emit("user:login", "张三")
// 输出:
// 发送欢迎邮件给用户: 张三
// 记录登录日志: 张三
```

### 4.3 责任链模式（Chain of Responsibility）

**意图**：将请求沿链传递，直到有对象处理它。

```go
// 处理器接口
type Handler interface {
    SetNext(handler Handler) Handler
    Handle(request string) string
}

// 基础处理器
type BaseHandler struct {
    next Handler
}

func (h *BaseHandler) SetNext(handler Handler) Handler {
    h.next = handler
    return handler
}

func (h *BaseHandler) Handle(request string) string {
    if h.next != nil {
        return h.next.Handle(request)
    }
    return ""
}

// 具体处理器
type AuthHandler struct {
    BaseHandler
}

func (a *AuthHandler) Handle(request string) string {
    if strings.Contains(request, "token") {
        fmt.Println("✓ 认证通过")
        return a.BaseHandler.Handle(request)
    }
    return "✗ 认证失败"
}

type RateLimitHandler struct {
    BaseHandler
    count int
}

func (r *RateLimitHandler) Handle(request string) string {
    r.count++
    if r.count > 100 {
        return "✗ 请求过于频繁"
    }
    fmt.Println("✓ 限流检查通过")
    return r.BaseHandler.Handle(request)
}

// 使用
auth := &AuthHandler{}
rateLimit := &RateLimitHandler{}
auth.SetNext(rateLimit)

result := auth.Handle("GET /api/data?token=abc")
```

### 4.4 命令模式（Command）

**意图**：将请求封装为对象，支持参数化和队列化。

```go
// 命令接口
type Command interface {
    Execute() string
    Undo() string
}

// 接收者
type TextEditor struct {
    content string
}

func (e *TextEditor) InsertText(text string) string {
    e.content += text
    return "插入文本: " + text
}

func (e *TextEditor) DeleteText(length int) string {
    if len(e.content) < length {
        length = len(e.content)
    }
    deleted := e.content[len(e.content)-length:]
    e.content = e.content[:len(e.content)-length]
    return "删除文本: " + deleted
}

// 具体命令
type InsertCommand struct {
    editor *TextEditor
    text   string
}

func (c *InsertCommand) Execute() string { return c.editor.InsertText(c.text) }
func (c *InsertCommand) Undo() string    { return c.editor.DeleteText(len(c.text)) }

// 调用者
type CommandHistory struct {
    history []Command
}

func (h *CommandHistory) Execute(cmd Command) string {
    result := cmd.Execute()
    h.history = append(h.history, cmd)
    return result
}

func (h *CommandHistory) Undo() string {
    if len(h.history) == 0 {
        return "没有可撤销的操作"
    }
    last := h.history[len(h.history)-1]
    h.history = h.history[:len(h.history)-1]
    return last.Undo()
}
```

### 4.5 状态模式（State）

**意图**：允许对象在内部状态改变时改变行为。

```go
// 状态接口
type OrderState interface {
    Next(order *Order) string
    Status() string
}

// 具体状态
type NewOrder struct{ order *Order }
func (s *NewOrder) Next(order *Order) string {
    order.state = &PaidOrder{order: order}
    return "订单已支付"
}
func (s *NewOrder) Status() string { return "新建" }

type PaidOrder struct{ order *Order }
func (s *PaidOrder) Next(order *Order) string {
    order.state = &ShippedOrder{order: order}
    return "订单已发货"
}
func (s *PaidOrder) Status() string { return "已支付" }

type ShippedOrder struct{ order *Order }
func (s *ShippedOrder) Next(order *Order) string {
    order.state = &DeliveredOrder{order: order}
    return "订单已签收"
}
func (s *ShippedOrder) Status() string { return "已发货" }

type DeliveredOrder struct{ order *Order }
func (s *DeliveredOrder) Next(order *Order) string {
    return "订单已完成，无法继续"
}
func (s *DeliveredOrder) Status() string { return "已签收" }

// 上下文
type Order struct {
    state OrderState
}

func NewOrder() *Order {
    o := &Order{}
    o.state = &NewOrder{order: o}
    return o
}

func (o *Order) Next() string { return o.state.Next(o) }
func (o *Order) Status() string { return o.state.Status() }
```

### 4.6 中介者模式（Mediator）

**意图**：用一个中介对象封装对象间的交互。

```go
// 中介者
type ChatRoom struct {
    users map[string]*User
}

func (c *ChatRoom) Register(user *User) {
    c.users[user.name] = user
}

func (c *ChatRoom) Send(from, to, message string) {
    if target, ok := c.users[to]; ok {
        target.Receive(from, message)
    }
}

// 同事类
type User struct {
    name string
    room *ChatRoom
}

func (u *User) Send(to, message string) {
    fmt.Printf("%s → %s: %s\n", u.name, to, message)
    u.room.Send(u.name, to, message)
}

func (u *User) Receive(from, message string) {
    fmt.Printf("%s ← %s: %s\n", u.name, from, message)
}
```

### 4.7 模板方法模式（Template Method）

**意图**：在父类中定义算法骨架，子类重写具体步骤。

```go
// 模板
type DataMiner interface {
    Mine() // 模板方法
    OpenFile()
    ExtractData()
    ParseData()
    AnalyzeData()
    SendReport()
    CloseFile()
}

// 基础实现
type AbstractDataMiner struct{}

func (a *AbstractDataMiner) Mine() {
    a.OpenFile()
    a.ExtractData()
    a.ParseData()
    a.AnalyzeData()
    a.SendReport()
    a.CloseFile()
}

// CSV 数据挖掘器
type CSVMiner struct{ AbstractDataMiner }

func (c *CSVMiner) OpenFile()    { fmt.Println("打开 CSV 文件") }
func (c *CSVMiner) ExtractData() { fmt.Println("提取 CSV 数据") }
func (c *CSVMiner) ParseData()   { fmt.Println("解析 CSV 格式") }
func (c *CSVMiner) AnalyzeData() { fmt.Println("分析 CSV 数据") }
func (c *CSVMiner) SendReport()  { fmt.Println("发送 CSV 报告") }
func (c *CSVMiner) CloseFile()   { fmt.Println("关闭 CSV 文件") }
```

## 五、设计模式选择指南

| 场景 | 推荐模式 |
|------|---------|
| 全局只需一个实例 | 单例 |
| 不知道具体类型时创建对象 | 工厂方法 / 抽象工厂 |
| 构建复杂对象 | 建造者 |
| 复用已有对象 | 原型 |
| 接口不兼容 | 适配器 |
| 动态添加功能 | 装饰器 |
| 控制对象访问 | 代理 |
| 简化子系统调用 | 外观 |
| 多个算法可切换 | 策略 |
| 状态驱动行为变化 | 状态 |
| 事件通知 | 观察者 |
| 请求处理链 | 责任链 |
| 命令排队/撤销 | 命令 |

## 六、设计原则

### SOLID 原则

| 原则 | 英文 | 说明 |
|------|------|------|
| 单一职责 | Single Responsibility | 一个类只有一个引起变化的原因 |
| 开闭原则 | Open/Closed | 对扩展开放，对修改关闭 |
| 里氏替换 | Liskov Substitution | 子类可以替换父类 |
| 接口隔离 | Interface Segregation | 接口要小而专 |
| 依赖倒置 | Dependency Inversion | 依赖抽象而非具体实现 |

### 其他原则

- **迪米特法则（LoD）**：最少知识原则，一个对象应该对其他对象有最少的了解
- **组合优于继承**：优先使用对象组合而非类继承
- **针对接口编程**：面向接口而非面向实现编程
