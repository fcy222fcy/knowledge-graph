## 第2页：项目背景与目标（~20秒）

**背景痛点（2×2 卡片）**：
- **知识获取低效**：海量课程资料分散无序，学生难以快速定位知识点
- **知识体系零散**：知识点间关联被隐藏，无法构建系统化网络
- **评估方式单一**：传统考试难以动态、个性化评估学习效果
- **资源更新滞后**：教师手动更新耗时，内容滞后于技术发展

**项目目标（3个卡片）**：
- **知识图谱可视化**：将课程文档转化为知识图谱，结构化存储与可视化展示
- **智能问答服务**：基于 RAG + 本地 LLM，提供精准自然语言问答
- **学习分析评估**：分析答题记录，自动评估知识点掌握度，生成可视化报告

**底部说明**：两类角色：学生端 + 教师端

## 第5页：核心功能 — 智能问答 RAG（~25秒）

**RAG 五步流程（横向流程卡片）**：
1. **用户提问** → 自然语言输入，支持多轮对话
2. **文本向量化** → Nomic-embed-text，768 维向量
3. **向量检索** → 余弦相似度，Top-K 匹配
4. **LLM 生成** → Qwen3:8b，检索增强生成
5. **流式输出** → SSE 推送，逐 Token 返回

**降级方案**：Ollama 不可用时自动切换关键词匹配，基于 KnowledgePoint 表模糊查询

**多轮对话**：AskSession 维护会话上下文，置信度评分展示回答可靠程度

## 第6页：核心功能 — 知识图谱 & 题库（~25秒）

**知识图谱构建与浏览**：
- 教师选择已审核文档，一键触发自动构建
- LLM Prompt 引导抽取知识点与关系（结构化 JSON）
- 降级方案：正则 + 软件工程本体关键词规则抽取
- 去重检查 → 写入 Neo4j → D3.js 力导向可视化
- 支持节点拖拽、滚轮缩放、点击查看详情

**题库练习**：
- 题目列表支持按难度、知识点筛选
- 单选/多选题型，难度分 easy/medium/hard
- 选择答案提交 → 即时评分 + 详细解析

**学习分析 & 教师端**：
- ECharts 统计图表：问答次数、正确率、薄弱知识点
- 教师端独立管理后台，资料审核、知识点/题目 CRUD、学生管理
