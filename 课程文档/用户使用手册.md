# SE智图问答平台 用户使用手册

## 1 引言

### 1.1 编写目的

本手册旨在为"SE智图问答"平台的用户提供全面的操作指导，帮助学生用户和教师用户快速掌握平台的各项功能，包括注册登录、智能问答、知识图谱浏览、资料管理、题库练习、学习分析以及教师端管理等操作。预期读者为使用本平台的软件工程课程学生和授课教师。

### 1.2 背景

- **软件系统名称**：SE智图问答平台（基于知识图谱的软件工程课程智能问答系统）
- **任务提出者**：软件工程课程教学团队
- **开发者**：符成岩、芮晓雨、苗琳
- **用户**：软件工程课程学生及授课教师
- **运行环境**：本地部署，通过浏览器访问

### 1.3 定义

| 术语 | 说明 |
|------|------|
| 知识图谱 | 以图结构表示知识及其关联关系的技术，节点表示知识点，边表示知识点间的关系 |
| RAG | 检索增强生成（Retrieval-Augmented Generation），结合检索与大语言模型生成回答的技术 |
| LLM | 大语言模型（Large Language Model），如本平台使用的 Qwen3:8b |
| MinIO | 开源对象存储系统，用于文件上传和存储 |
| JWT | JSON Web Token，用于用户身份认证的令牌机制 |
| Neo4j | 图数据库，用于存储知识图谱的节点和关系 |

## 2 用途

### 2.1 功能

本平台面向软件工程课程教学，提供以下核心功能：

| 功能模块 | 适用角色 | 功能说明 |
|----------|----------|----------|
| 用户注册与登录 | 学生、教师 | 支持用户名+密码注册登录，区分学生和教师角色 |
| 智能问答 | 学生 | 基于知识图谱的自然语言问答，支持流式输出和历史记录 |
| 知识图谱浏览 | 学生 | 可视化展示知识点及其关联关系，支持搜索和筛选 |
| 资料管理 | 学生、教师 | 上传和管理课程资料（PDF、Word、PPT、Markdown等） |
| 题库练习 | 学生 | 基于课程知识的题目练习，支持自动评分和解析 |
| 学习分析 | 学生 | 学习时长统计、问答次数统计、薄弱知识点分析 |
| 资料审核 | 教师 | 审核学生上传的课程资料 |
| 知识点管理 | 教师 | 管理知识图谱中的节点和关系 |
| 题目管理 | 教师 | 创建和管理题库中的题目 |
| 学生管理 | 教师 | 查看和管理学生账户信息 |

**功能极限范围**：

- 问答功能基于本地部署的 Qwen3:8b 模型，回答质量受模型参数限制
- 知识图谱构建依赖上传的课程资料质量，资料内容越丰富，图谱越完整
- 当前为本地化部署，仅支持局域网或单机访问，不支持大规模并发

### 2.2 性能

#### 2.2.1 精度

- 问答系统的回答准确性取决于课程资料的完整性和知识图谱的构建质量
- 语义检索采用向量相似度匹配，召回率与文本分块策略相关
- 题库练习的评分基于预设的标准答案进行匹配

#### 2.2.2 时间特性

| 操作 | 响应时间 |
|------|----------|
| 页面加载 | ≤ 2 秒 |
| 用户登录/注册 | ≤ 1 秒 |
| 智能问答响应 | ≤ 5 秒（流式输出首 token ≤ 2 秒） |
| 知识图谱加载 | ≤ 3 秒（取决于节点数量） |
| 文件上传 | 取决于文件大小和网络环境 |
| 知识图谱构建 | 取决于文档大小，通常 10-60 秒 |

#### 2.2.3 灵活性

- 支持多种文档格式上传（PDF、DOCX、PPTX、Markdown、TXT）
- 知识图谱支持搜索、筛选、缩放等交互操作
- 问答支持任意自然语言输入，不限定固定提问格式
- 教师端可灵活管理知识点和题目内容

### 2.3 安全保密

- **身份认证**：采用 JWT（JSON Web Token）进行用户身份验证，令牌有效期 24 小时
- **权限控制**：基于角色的访问控制（RBAC），学生和教师权限严格隔离
- **密码安全**：用户密码使用加密存储，不以明文形式保存
- **教师鉴权**：教师端接口通过独立的 `RequireTeacherAuth` 中间件保护，仅持有教师身份的用户可访问管理功能
- **文件安全**：上传文件存储于 MinIO 对象存储，通过独立服务隔离，防止直接文件系统访问

## 3 运行环境

### 3.1 硬设备

**最小配置要求**：

| 项目 | 要求 |
|------|------|
| 处理器 | Intel Core i5 或同等性能 CPU（建议 4 核以上） |
| 内存 | 8GB RAM（建议 16GB，运行 Ollama 推理时需更多内存） |
| 硬盘 | 20GB 可用磁盘空间 |
| 网络 | 无特殊要求（本地部署，仅需本机访问） |

**推荐配置**：

| 项目 | 推荐 |
|------|------|
| 处理器 | Intel Core i7 或同等性能 CPU |
| 内存 | 16GB RAM 以上 |
| 硬盘 | 50GB SSD |
| GPU | 支持 CUDA 的 NVIDIA 显卡（可选，用于加速 LLM 推理） |

### 3.2 支持软件

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| 操作系统 | Windows 10/11 64位 | 主要支持平台 |
| Go | 1.25+ | 后端编译运行环境 |
| Node.js | 18+ | 前端构建环境 |
| MySQL | 8.0+ | 业务数据存储 |
| Neo4j | 5.x | 知识图谱数据存储 |
| MinIO | 最新版 | 对象存储服务 |
| Ollama | 最新版 | 本地 LLM 推理引擎 |
| 现代浏览器 | Chrome / Edge / Firefox | 用户访问界面 |

### 3.3 数据结构

本平台涉及以下核心数据存储：

| 数据库 | 用途 | 存储内容 |
|--------|------|----------|
| MySQL | 业务数据 | 用户信息、资料元数据、题目、问答记录、学习统计数据 |
| Neo4j | 知识图谱 | 知识点节点、知识点关系、节点属性 |
| MinIO | 文件存储 | 上传的课程资料文件（PDF、Word 等） |

## 4 使用过程

### 4.1 安装与初始化

#### 4.1.1 环境准备

在首次使用前，需要确保以下服务已安装并正常运行：

**步骤 1：安装 MySQL**

1. 下载并安装 MySQL 8.0 或更高版本
2. 创建数据库：`CREATE DATABASE se_knowledge_qa CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
3. 记录数据库连接信息（主机、端口、用户名、密码）

**步骤 2：安装 Neo4j**

1. 下载并安装 Neo4j 5.x 社区版
2. 启动 Neo4j 服务
3. 通过浏览器访问 `http://localhost:7474`，默认用户名/密码为 `neo4j`/`neo4j`，首次登录后修改密码
4. 记录 Bolt 连接地址（默认 `bolt://localhost:7687`）

**步骤 3：安装 MinIO**

1. 下载 MinIO Server 可执行文件
2. 启动 MinIO：`minio server ./miniodata`
3. 通过浏览器访问 `http://localhost:9000`，使用默认凭证 `minioadmin`/`minioadmin` 登录
4. 创建存储桶（Bucket），名称与配置文件中一致

**步骤 4：安装 Ollama**

1. 从 https://ollama.com 下载并安装 Ollama
2. 拉取所需模型：`ollama pull qwen3:8b`（问答模型）和 `ollama pull nomic-embed-text`（向量化模型）
3. 确认 Ollama 服务运行在 `http://localhost:11434`

#### 4.1.2 配置项目

1. 在后端目录创建 `.env` 文件，配置数据库连接等参数：

```env
# 服务端口
SERVER_PORT=8081

# MySQL 配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=se_knowledge_qa

# Neo4j 配置
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_password

# MinIO 配置
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=documents

# Ollama 配置
OLLAMA_BASE_URL=http://localhost:11434
```

#### 4.1.3 一键启动

双击运行项目根目录下的 `start.bat` 脚本，将自动启动以下服务：

- Neo4j 图数据库（端口 7474/7687）
- Go 后端服务（端口 8080）
- 前端开发服务器（端口 5173）

启动完成后，打开浏览器访问 `http://localhost:5173` 即可使用平台。

停止所有服务可运行 `stop.bat`。

#### 4.1.4 学生用户注册与登录

1. 打开浏览器，访问 `http://localhost:5173`
2. 系统自动跳转至登录页面
3. 首次使用点击"注册"链接，进入注册页面
4. 填写用户名、密码、确认密码、邮箱，点击"注册"按钮
5. 注册成功后自动跳转至登录页面
6. 输入用户名和密码，点击"登录"按钮
7. 登录成功后进入学生端首页

#### 4.1.5 教师用户登录

1. 打开浏览器，访问教师端地址（端口与学生端不同，详见部署配置）
2. 在登录页面输入教师账号和密码
3. 点击"登录"按钮
4. 登录成功后进入教师管理后台仪表盘

### 4.2 学生功能

#### 4.2.1 首页仪表盘

登录后进入首页，展示以下信息：

- **学习概况**：今日学习时长、累计问答次数、已练习题目数
- **最近问答**：最近的问答记录快捷入口
- **学习趋势**：近 7 天学习时长柱状图
- **快捷入口**：各功能模块的快速导航卡片

#### 4.2.2 智能问答

**操作步骤**：

1. 点击左侧导航栏"问答中心"进入问答页面
2. 在输入框中输入问题，例如："什么是设计模式？"
3. 点击"发送"按钮或按回车键提交问题
4. 系统将基于知识图谱和课程资料进行检索，流式输出回答
5. 回答下方展示"知识来源"和"相关知识点"，点击可查看详情
6. 左侧会话列表记录历史问答，可点击切换或新建会话

**使用技巧**：

- 问题越具体，回答越精准。例如"单例模式的实现方式有哪些？"比"设计模式"效果更好
- 可参考回答下方的"相关知识点"进行延伸学习
- 历史会话支持查看，方便复习之前的问题

#### 4.2.3 知识图谱浏览

**操作步骤**：

1. 点击左侧导航栏"知识图谱"进入图谱页面
2. 页面展示知识图谱的力导向布局，节点表示知识点，边表示关联关系
3. **拖拽节点**：按住鼠标左键拖拽节点，调整布局
4. **缩放**：鼠标滚轮缩放图谱
5. **点击节点**：点击任意节点查看该知识点的详细信息（名称、类型、描述、关联知识点）
6. **搜索筛选**：在搜索框输入关键词，高亮匹配的知识点节点
7. **筛选**：通过类型筛选器筛选特定类别的知识点

#### 4.2.4 资料管理

**上传资料**：

1. 点击左侧导航栏"资料管理"进入资料页面
2. 点击"上传资料"按钮
3. 选择要上传的文件（支持 PDF、DOCX、PPTX、Markdown、TXT 格式）
4. 确认上传，系统将自动存储文件并记录元信息
5. 上传完成后，资料状态显示为"待审核"

**查看资料**：

1. 在资料列表中可查看已上传的资料信息（文件名、类型、大小、上传时间、状态）
2. 点击"下载"可下载资料文件
3. 状态为"已审核"的资料可被用于知识图谱构建

#### 4.2.5 题库练习

**操作步骤**：

1. 点击左侧导航栏"题库练习"进入练习页面
2. 选择题目类型（选择题、填空题、简答题等）和难度
3. 系统展示题目列表，点击"开始练习"
4. 逐题作答，选择题点击选项，填空题输入答案，简答题填写文字
5. 完成后点击"提交"，系统自动评分并展示解析
6. 查看得分和错题解析，巩固薄弱知识点

#### 4.2.6 学习分析

**操作步骤**：

1. 点击左侧导航栏"学习分析"进入统计页面
2. 页面展示以下统计数据：
   - **学习时长统计**：本周/本月每日学习时长柱状图
   - **问答统计**：累计问答次数、本周新增问答数
   - **薄弱知识点**：根据答题数据识别的薄弱知识点列表
   - **学习趋势**：近 7 天学习趋势折线图
3. 点击薄弱知识点可跳转至问答中心进行针对性学习

### 4.3 教师功能

#### 4.3.1 管理仪表盘

教师登录后进入管理后台仪表盘，展示系统概况：

- **学生总数**：当前注册学生数量
- **资料总数**：已上传课程资料数量
- **题目总数**：题库中题目数量
- **问答总数**：学生问答记录总数
- **待审核资料**：等待审核的资料数量

#### 4.3.2 资料审核

**操作步骤**：

1. 点击左侧导航栏"资料审核"进入审核页面
2. 查看待审核资料列表，显示文件名、上传者、上传时间
3. 点击"查看"可预览资料内容
4. 确认资料质量后，点击"通过"完成审核
5. 如资料不符合要求，点击"拒绝"并填写拒绝原因
6. 审核通过的资料将被纳入知识图谱构建的数据源

#### 4.3.3 知识点管理

**操作步骤**：

1. 点击左侧导航栏"知识点管理"进入管理页面
2. 查看当前知识图谱中的所有知识点节点
3. **新增知识点**：点击"新增"按钮，填写节点名称、类型、描述等信息，保存
4. **编辑知识点**：点击节点行的"编辑"按钮，修改节点信息
5. **删除知识点**：点击"删除"按钮，确认后删除节点及其关联关系
6. **搜索筛选**：通过关键词或类型筛选知识点

#### 4.3.4 题目管理

**操作步骤**：

1. 点击左侧导航栏"题目管理"进入管理页面
2. 查看题库中的所有题目
3. **新增题目**：点击"新增"按钮，选择题目类型，填写题干、选项（选择题）、正确答案、解析、难度等级
4. **编辑题目**：点击题目行的"编辑"按钮，修改题目内容
5. **删除题目**：点击"删除"按钮，确认后删除
6. **批量操作**：支持批量删除、批量设置难度

#### 4.3.5 学生管理

**操作步骤**：

1. 点击左侧导航栏"学生管理"进入管理页面
2. 查看所有注册学生的信息列表（用户名、邮箱、注册时间）
3. 可通过搜索框按用户名筛选学生
4. 点击学生行可查看该学生的详细学习数据（学习时长、问答次数、答题情况）

### 4.4 出错处理和恢复

| 错误场景 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 登录失败，提示"用户不存在" | 用户名输入错误或未注册 | 检查用户名拼写，或点击"注册"创建新账号 |
| 登录失败，提示"密码错误" | 密码输入错误 | 重新输入密码，注意大小写 |
| 问答无响应或超时 | Ollama 服务未启动或模型未加载 | 检查 Ollama 是否运行，执行 `ollama list` 确认模型已安装 |
| 文件上传失败 | 文件格式不支持或文件过大 | 确认文件格式为 PDF/DOCX/PPTX/MD/TXT，文件大小不超过限制 |
| 知识图谱加载空白 | Neo4j 服务未启动或无数据 | 检查 Neo4j 服务状态，确保已上传资料并执行知识图谱构建 |
| 页面加载缓慢 | 服务未完全启动或资源不足 | 等待服务完全启动，检查系统内存和 CPU 使用情况 |
| 前端页面白屏 | 前端服务未启动或编译错误 | 检查前端开发服务器是否运行，查看终端错误日志 |
| 知识图谱构建失败 | Ollama 向量模型未加载 | 检查 Ollama 是否运行，执行 `ollama pull nomic-embed-text` 加载向量模型 |
| 数据库连接失败 | MySQL 未启动或配置错误 | 检查 MySQL 服务状态，核对 `.env` 中的数据库配置 |

**恢复建议**：

1. 遇到服务无法访问时，先运行 `stop.bat` 停止所有服务，再运行 `start.bat` 重新启动
2. 查看各服务的终端输出日志，定位具体错误信息
3. 确认所有依赖服务（MySQL、Neo4j、MinIO、Ollama）均已正常启动
4. 如问题持续，检查端口是否被占用（使用 `netstat -ano | findstr 端口号`）
