# 基于知识图谱的软件工程课程问答平台 用户使用手册

---

## 1 引言

### 1.1 编写目的

本手册旨在为 基于知识图谱的软件工程课程问答平台的用户提供全面的操作指导，帮助用户快速掌握系统的各项功能。本手册适用于以下读者：

- **学生用户**：了解如何使用平台进行学习、提问、答题和查看学习统计
- **教师用户**：了解如何管理资料、知识点、作业和学生信息
- **系统管理员**：了解系统的整体架构和维护操作

### 1.2 背景

**软件系统名称**：基于知识图谱的软件工程课程问答平台

本平台是一个基于知识图谱的软件工程课程智能问答与学习系统，旨在为学生提供智能化的学习辅助服务，帮助学生更好地理解软件工程相关知识。

**项目相关方**：
- 任务提出者：软件工程课程教学团队
- 开发者：软件工程项目组
- 主要用户：软件工程课程学生、授课教师

### 1.3 定义

| 术语 | 定义 |
|------|------|
| 知识图谱 | 以图结构组织知识点及其关系的知识表示方法 |
| 问答会话 | 用户与系统进行的一轮或多轮问答交互 |
| 知识点 | 软件工程课程中的基本概念、原理或方法 |
| 作业 | 教师布置的测验题目集合 |
| 掌握度 | 系统根据答题情况计算的用户对某知识点的掌握程度 |

### 1.4 参考资料

- 《软件工程》课程教学大纲
- 系统需求规格说明书
- 系统设计文档

## 2 用途

### 2.1 功能

基于知识图谱的软件工程课程问答平台提供以下核心功能：

#### 学生端功能

| 功能模块 | 功能说明 | 使用场景 |
|----------|----------|----------|
| 智能问答 | 基于知识图谱的AI智能问答，支持流式响应 | 学习过程中遇到问题时提问 |
| 知识图谱 | 可视化展示知识点及其关系，支持搜索和浏览 | 查看知识结构，理解知识点关联 |
| 资料管理 | 上传、查看、管理学习资料 | 上传课件、笔记等学习资料 |
| 答题中心 | 完成教师布置的作业，查看答题结果 | 完成课后作业、自我测试 |
| 分析统计 | 查看学习时长、提问数、正确率等统计信息 | 了解自身学习情况 |
| 首页概览 | 展示今日统计、快速操作入口、最近问答 | 快速了解学习状态 |

#### 教师端功能

| 功能模块 | 功能说明 | 使用场景 |
|----------|----------|----------|
| 仪表盘 | 展示系统整体数据概览 | 查看班级整体学习情况 |
| 资料审核 | 审核学生上传的学习资料 | 审核并管理教学资料 |
| 知识点管理 | 管理知识图谱中的知识点 | 维护和更新知识库 |
| 作业管理 | 创建、发布和管理作业 | 布置和管理课后作业 |
| 学生管理 | 查看和管理学生信息 | 了解学生学习情况 |

### 2.2 性能

#### 2.2.1 精度

- 问答响应准确率：基于知识图谱的检索，答案来源可追溯
- 学习统计精度：学习时长精确到分钟，正确率精确到百分比
- 知识点匹配精度：支持关键词模糊搜索

#### 2.2.2 时间特性

| 操作类型 | 响应时间 |
|----------|----------|
| 页面加载 | ≤ 2秒 |
| 智能问答（首字） | ≤ 3秒 |
| 智能问答（完整） | ≤ 30秒 |
| 知识图谱渲染 | ≤ 5秒 |
| 文件上传 | 视文件大小，一般 ≤ 10秒 |
| 数据统计查询 | ≤ 3秒 |

#### 2.2.3 灵活性

- 支持主流浏览器（Chrome、Firefox、Edge、Safari）
- 响应式设计，适配桌面端和移动端
- 支持多种题型（单选、多选、判断）
- 问答支持多轮对话，上下文关联

### 2.3 安全保密

- 采用 JWT Token 认证机制，保障用户身份安全
- 密码采用 bcrypt 加密存储
- 敏感操作需身份验证
- 学生仅能访问自己的学习数据
- 教师仅能管理其权限范围内的数据

## 3 运行环境

### 3.1 硬设备

**客户端（最低配置）**：
- 处理器：Intel Core i3 或同等性能
- 内存：4GB RAM
- 硬盘：1GB 可用空间
- 网络：稳定的互联网连接（建议带宽 ≥ 2Mbps）

**服务器端**：
- 处理器：Intel Xeon 或同等性能，4核以上
- 内存：8GB RAM 以上
- 硬盘：50GB 以上 SSD
- 网络：100Mbps 以上

### 3.2 支持软件

**客户端**：
- 操作系统：Windows 10+、macOS 10.15+、Linux
- 浏览器：Chrome 90+、Firefox 88+、Edge 90+、Safari 14+

**服务端**：
- 操作系统：Linux (Ubuntu 20.04+)
- 运行环境：Go 1.21+
- 数据库：MySQL 8.0+、Neo4j 4.4+
- 对象存储：MinIO
- AI模型：Ollama (本地部署)

### 3.3 数据结构

系统使用以下数据存储：

| 存储类型 | 用途 |
|----------|------|
| MySQL | 存储用户信息、问答记录、作业数据、文档元数据 |
| Neo4j | 存储知识图谱（节点和关系） |
| MinIO | 存储上传的文档文件 |

## 4 使用过程

### 4.1 安装与初始化

#### 4.1.1 环境准备

1. **安装 Go 环境**
   ```bash
   # 下载并安装 Go 1.21+
   # 验证安装
   go version
   ```

2. **安装 MySQL**
   ```bash
   # 安装 MySQL 8.0
   # 创建数据库
   mysql -u root -p -e "CREATE DATABASE se_intelligence;"
   ```

3. **安装 Neo4j**
   ```bash
   # 安装 Neo4j 4.4+
   # 启动服务并设置密码
   ```

4. **安装 MinIO**
   ```bash
   # 下载并启动 MinIO
   minio server /data --console-address ":9001"
   ```

5. **安装 Ollama**
   ```bash
   # 安装 Ollama 并拉取模型
   ollama pull qwen2.5
   ```

#### 4.1.2 后端服务启动

```bash
# 进入后端目录
cd software engineering-back

# 安装依赖
go mod tidy

# 启动服务
go run cmd/server/main.go
```

服务默认运行在 `http://localhost:8080`

#### 4.1.3 前端服务启动

```bash
# 进入前端目录（学生端）
cd software engineering-fronted/frontend-vue-app

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

学生端默认运行在 `http://localhost:5173`

```bash
# 进入教师端目录
cd software engineering-fronted/admin-vue-app

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

教师端默认运行在 `http://localhost:5174`

### 4.2 学生功能

#### 4.2.1 注册账号

1. 打开浏览器，访问学生端地址
2. 点击登录页面底部的 **"立即注册"** 链接

![登录页面](images/student-login.png)

3. 在注册页面填写以下信息：
   - 用户名：4-20位字母、数字或下划线
   - 密码：6-20位字符
   - 确认密码：再次输入密码

![注册页面](images/student-register.png)

4. 点击 **"注册"** 按钮完成注册
5. 注册成功后自动跳转到登录页面

#### 4.2.2 登录系统

1. 在登录页面输入用户名和密码
2. 可勾选 **"记住我"** 选项，下次自动登录
3. 点击 **"登录"** 按钮

![登录操作](images/student-login.png)

4. 登录成功后跳转到首页

#### 4.2.3 首页概览

登录后默认显示首页，包含以下模块：

![首页界面](images/student-home.png)

**今日统计卡片**：
- 今日学习时长
- 今日提问数
- 总学习时长
- 总提问数

**快速操作**：
- 智能问答：快速进入问答中心
- 知识图谱：查看知识结构
- 答题中心：完成作业

**最近问答**：显示最近的问答记录

**学习趋势**：近7天的学习数据趋势图

#### 4.2.4 智能问答

1. 点击左侧导航栏的 **"问答中心"** 或首页的 **"智能问答"** 快速入口

2. 进入问答页面，左侧为会话列表，右侧为对话区域

![问答中心界面](images/student-qa.png)

3. **发起提问**：
   - 在底部输入框输入问题
   - 点击发送按钮或按 Enter 键
   - 系统将流式返回回答

![提问操作](images/student-qa-source.png)

4. **查看来源引用**：
   - 回答下方会显示参考来源
   - 点击可查看原始文档位置

![来源引用](images/student-qa-source.png)

5. **管理会话**：
   - 点击 **"新对话"** 创建新的问答会话
   - 在左侧列表切换不同的会话
   - 每个会话独立保存对话历史

#### 4.2.5 知识图谱

1. 点击左侧导航栏的 **"知识图谱"**

2. 进入知识图谱页面，显示可视化的知识网络

![知识图谱界面](images/student-graph.png)

3. **浏览图谱**：
   - 鼠标滚轮缩放图谱
   - 拖拽画布移动视图
   - 点击节点查看详情

4. **搜索知识点**：
   - 在顶部搜索框输入关键词
   - 图谱将高亮匹配的节点

![搜索知识点](images/student-graph-search.png)

5. **查看节点详情**：
   - 点击任意节点，右侧弹出详情面板
   - 显示节点名称、类型、描述等信息
   - 可跳转到相关文档

![节点详情](images/student-graph-detail.png)

6. **构建图谱**（教师权限）：
   - 点击工具栏的 **"构建图谱"** 按钮
   - 选择要处理的文档
   - 系统自动提取知识点并生成图谱

#### 4.2.6 资料管理

1. 点击左侧导航栏的 **"资料管理"**

2. 进入资料管理页面，显示文档列表

![资料管理界面](images/student-files.png)

3. **上传文档**：
   - 点击 **"上传文档"** 按钮
   - 选择本地文件（支持 PDF、Word、Markdown）
   - 点击上传，等待处理完成

![上传文档](images/student-files-upload.png)

4. **搜索和筛选**：
   - 在搜索框输入关键词搜索文档
   - 使用状态下拉框筛选（待处理/处理中/已完成/失败）

5. **查看文档详情**：
   - 点击文档名称或 **"查看"** 按钮
   - 弹窗显示文档详细信息

![文档详情](images/student-files-detail.png)

6. **删除文档**：
   - 点击 **"删除"** 按钮
   - 确认删除后文档将被移除

#### 4.2.7 答题中心

1. 点击左侧导航栏的 **"答题"**

2. 进入答题中心，显示作业列表

![答题中心界面](images/student-quiz.png)

3. **查看作业**：
   - 每个作业卡片显示标题、题数、总分、截止时间
   - 已完成的作业显示得分

4. **开始答题**：
   - 点击 **"去答题"** 按钮进入答题页面

![答题界面](images/student-quiz-answer.png)

5. **作答**：
   - 单选题：点击选择一个选项
   - 多选题：点击选择多个选项
   - 判断题：选择对或错

6. **提交作业**：
   - 答完后点击 **"提交作业"** 按钮
   - 确认提交后查看得分

![作业结果](images/student-quiz-result.png)

7. **查看解析**：
   - 提交后可查看每道题的答案和解析
   - 正确的题目显示绿色边框
   - 错误的题目显示红色边框

#### 4.2.8 分析统计

1. 点击左侧导航栏的 **"分析统计"**

2. 进入统计页面，查看学习数据分析

![分析统计界面](images/student-stats.png)

3. **总览数据**：
   - 总学习时长
   - 总提问数
   - 总测验数
   - 平均正确率
   - 知识点掌握进度

4. **知识点掌握情况**：
   - 雷达图展示各知识点掌握度
   - 颜色区分已掌握/学习中/薄弱

![掌握情况雷达图](images/student-stats.png)

5. **热门知识点排行**：
   - 显示最常被问到的知识点

6. **薄弱知识点**：
   - 列出正确率较低的知识点
   - 提供相关练习题建议

![薄弱知识点](images/student-stats.png)

7. **学习趋势**：
   - 折线图展示近7天学习数据变化

### 4.4 出错处理和恢复

#### 常见错误及处理方法

| 错误情况 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 登录失败 | 密码错误或账号不存在 | 检查用户名密码，或重新注册 |
| 问答无响应 | 网络问题或服务异常 | 检查网络连接，稍后重试 |
| 上传失败 | 文件过大或格式不支持 | 检查文件大小（≤50MB）和格式 |
| 页面空白 | 浏览器兼容性问题 | 使用 Chrome/Firefox 最新版 |
| 数据加载失败 | 网络超时 | 刷新页面重试 |

#### 系统恢复

1. **会话过期**：系统会自动跳转到登录页，重新登录即可
2. **网络中断**：恢复网络后刷新页面
3. **服务重启**：等待服务启动完成后刷新页面
