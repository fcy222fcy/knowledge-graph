基于知识图谱的软件工程课程问答平台 - 需求分析报告

1. 项目概述
软件工程作为一门实践性极强的学科，其知识体系具有高度的抽象性和关联性。传统教学材料主要以线性文本形式呈现，知识点之间的逻辑关系隐含在文本描述中，学生难以系统性地把握整体知识框架。本项目旨在利用知识图谱技术构建软件工程课程的结构化知识表示，实现知识点的显式关联与语义组织。
项目名称：基于知识图谱的软件工程课程智能问答平台
项目价值体现在以下三个核心维度：
知识图谱化：将课程中的核心概念、设计模式、架构原则等实体化对象抽取出来，建立节点与节点之间的语义连接。例如，将"设计模式"、"重构"、"单一职责原则"等概念实体关联，形成概念拓扑结构。这种结构化表示能够将隐性知识显性化，帮助学生建立系统化的认知模型。
智能检索：通过向量空间模型与图推理相结合的混合检索机制，实现基于语义理解的问答服务。用户输入自然语言查询后，系统基于课程材料进行精准定位与解答，相比传统搜索引擎具有更强的领域针对性和答案可靠性。
自动化测评：基于知识图谱的结构化特征，自动生成覆盖不同认知层次（记忆、理解、应用、分析、评价、创造）的题目类型，包括选择题、简答题、应用题等，既减轻教师重复性工作负担，又保证题目的知识覆盖率和认知层次多样性。
目标用户群体主要包含两类：
学生用户群体面临大量专业术语与抽象概念的理解挑战，通过该平台可实现知识点查询、概念阐释与示例演示的快速定位，支持预习复习与深度学习。
教师与管理员群体需要完成课程内容资源管理、题库构建与教学效果监测。通过答题数据分析可以识别学生的知识薄弱点，为教学策略调整提供数据支撑。
鉴于项目开发周期的限制性，本阶段采用本地化部署架构，优先验证核心功能的可行性与稳定性，后续迭代可扩展至分布式服务器环境部署。
2. 功能需求
2.1 用户角色与权限体系
基于RBAC（Role-Based Access Control）模型划分用户权限，实现细粒度的功能访问控制。系统采用三权分立的管理模式，保障用户操作的合法性与安全性。
表 1 系统用户角色与权限对照表
用户角色	权限级别	核心功能域
学生	查询者	语义检索、知识图谱浏览、题目练习、学习分析
教师	资源管理者	课程资源管理、题库建设、题目审核发布、学习数据监测
管理员	系统运维者	组织架构管理、权限分配、系统配置、监控审计
 2.2 核心功能模块架构
系统采用微服务架构设计，各功能模块松耦合、高内聚，具备良好的可扩展性与可维护性。
模块一：知识资源管理
课程资源管理系统是平台的底层数据支撑模块，承担非结构化文档到结构化知识的转换任务。软件工程课程资料通常包含PDF讲义、Word文档、PPT演示文稿等多格式内容，该模块需实现多格式文件的标准化处理流程。
功能描述：支持课程资源上传、NLP处理、知识抽取与图谱构建
具体包含以下功能组件：
文件上传服务：支持PDF、DOCX、Markdown、TXT等主流文档格式的批量上传，实现分片上传与断点续传机制，提升大文件传输的稳定性与用户体验。上传过程中提供实时进度反馈与错误状态码解析。
文档解析引擎：实现文档内容的结构化提取与智能分块。采用分词算法识别语义单元，结合段落结构、章节层级、内容相关性等多维度指标实施智能分块策略，保证知识单元的语义完整性。同时保留标题层级、列表结构等文档格式信息，维持原文的上下文关联。
知识抽取管道：基于NLP技术实现实体识别、关系抽取、属性提取的三级知识提取流程。实体识别模块识别术语、概念、方法论等关键名词实体；关系抽取模块构建实体间的语义关系网络；属性提取模块采集实体的定义、特征、示例等元数据信息。
知识图谱构建：将抽取的实体与关系映射为图数据库节点与边，采用Cypher查询语言进行图谱存储与索引优化。同时生成文本语义向量，通过向量化模型将知识节点映射到向量空间，为后续的语义检索提供基础支撑。
数据模型定义：采用面向对象的存储结构，文档实体包含元数据（ID、名称、版本、上传时间、处理状态）与内容分块集合。分块实体维护文本内容、向量索引ID、包含实体集合等关联信息。实体节点存储名称、类型、属性字典等图谱属性，实现灵活且完整的数据表示。
模块二：智能问答检索
语义检索服务是平台的核心价值实现模块，通过混合检索架构提供精准的知识问答服务。用户输入自然语言查询后，系统通过多模态检索机制定位目标知识内容。
功能描述：自然语言问答、语义检索、知识图谱推理、答案合成
查询处理：支持自然语言输入处理，实现查询意图识别与关键信息提取。提供话题标签辅助检索，提升检索的精确度与响应速度。查询历史记录功能支持用户回顾与二次检索。
混合检索策略：采用向量检索与图推理相结合的复合检索模式。向量相似度匹配基于余弦相似度或欧氏距离计算，在向量空间中快速定位语义相近的知识片段；图谱推理则通过路径查找与邻居节点扩展，揭示隐含的知识关联，提供更丰富的上下文信息。双重检索结果通过重排序算法进行融合，输出最优答案序列。
答案呈现：提供结构化答案展示，包含参考答案、引用来源片段、置信度评分、相关知识图谱可视化节点。支持相似问题推荐与知识关联展示，辅助用户构建完整的知识网络。答案来源可追溯，确保内容的可信度与可验证性。
数据模型：问答会话记录包含会话ID、用户ID、查询文本、检索策略参数、答案序列（含片段、置信度、来源）、相关节点集合、创建时间等元数据。这些数据集可用于学习行为分析与知识掌握度评估，为个性化学习路径推荐提供数据支撑。
模块三：自适应题库生成
智能题库生成模块通过知识图谱的图结构特征，实现基于认知层次的多维度题目生成。该模块旨在降低教师重复性工作负担，同时保证题目的认知覆盖度与质量一致性。
功能描述：认知驱动的题目自动生成、质量审核机制、题库智能管理
生成策略：支持多种题目类型与认知层次覆盖。
简答题：考察对概念内涵与外延的深度理解，要求对核心概念进行阐释与辨析
选择题：测试概念辨析与细节把握能力，通过选项干扰项设计考察认知精确度
填空题：强化术语记忆与知识点的熟练度
应用题：基于真实场景模拟，考察知识点的综合运用与问题解决能力
 
质量控制流程：自动生成的题目经过人工审核流程，审核维度包括题目表述清晰度、答案准确性、认知层次合理性、干扰项设计有效性等。审核通过后标记为已发布状态，支持题库的版本管理与迭代优化。
题库管理子系统：提供题目的检索、筛选、统计与分发功能。支持按课程、认知层次、知识点、难度系数等多维度筛选，可视化展示题库结构与分布。支持难度分级管理，适配不同层次学习者的学习需求。
 
数据模型设计：题目实体包含题目类型、题干内容、选项集合（选择题）、标准答案、解析说明、难度等级、认知层次标签、所属课程、题目状态（草稿/已发布）、生成方式（自动/人工）、关联知识节点集合等属性。该数据结构支持题目自动生成、人工编辑与版本管理的无缝集成，具备良好的扩展性。
模块四：学习效果评估
学习评估系统提供诊断性的学习反馈与多维度的学习行为分析，帮助学生识别知识薄弱点，优化学习策略，提升学习效能。
功能描述：在线答题评测、智能评分、学习诊断、统计分析
答题交互组件：实现题目展示、答案提交、即时评分反馈的完整交互流程。记录答题时间、答题路径等行为数据，为学习分析提供数据基础。
评分反馈机制：基于题目标准答案与评分规则实施自动化评分，结合知识图谱关联分析学生的知识掌握度。提供详细的错误解析与正确思路指引，实现知识点的精准定位与修正建议。
学习诊断分析：通过知识图谱可视化展示学生的知识点掌握分布与关联强度，识别认知断层与薄弱区域。基于题型的错误率分析识别认知模式与学习偏好，生成个性化的学习改进建议。
学习行为分析：提供学习统计仪表盘，展示知识点掌握度热力图、题型能力曲线、错题分布统计、学习进度追踪等可视化指标。错题集功能支持按知识点、错误率、时间等多维度筛选，辅助针对性复习。
数据模型设计：答题记录实体包含用户ID、题目ID、答案内容、作答状态、得分、反馈信息、知识节点关联、作答时长等属性。学习统计聚合实体包含知识点掌握度评分矩阵、题型正确率统计、错题索引、学习进度轨迹等维度，为学习效果评估提供结构化数据支撑。 
模块五：管理控制台
管理控制台为系统管理员与课程教师提供统一的后台管理界面，实现课程资源的全生命周期管理、用户权限体系维护与系统运行参数配置。
功能描述：组织架构管理、课程资源管控、权限体系配置、系统监控
课程资源管理：支持课程创建、课程元数据配置、章节结构编排、资源附件管理等操作。实现课程的访问控制策略配置，包括用户白名单管理、权限级别分配、课程可见性控制等，保障课程内容的版权安全与访问合规。
用户与权限管理：提供用户账号的全生命周期管理，包括用户创建、信息维护、账户封禁等操作。基于RBAC模型实现细粒度的权限分配，支持角色-权限的动态配置，满足不同组织架构与业务场景的权限管控需求。
系统参数配置：提供AI服务配置（模型参数、超时策略）、知识图谱构建参数（实体抽取阈值、关系抽取阈值）、向量检索参数（维度配置、相似度阈值）等系统级配置项。配置项变更实时生效，支持配置版本的版本管理与回滚机制。
 
3. 技术架构设计
系统采用前后端分离的微服务架构模式，前端负责用户界面展示与交互，后端提供RESTful API接口，Python AI服务专门处理NLP与知识图谱构建任务。数据持久层采用图数据库与向量数据库的组合存储方案，实现结构化与非结构化知识的统一管理。
3.1 整体架构
架构采用六边形架构理念，实现业务逻辑与基础设施的解耦。前端层专注UI渲染与用户交互，后端层处理业务逻辑与数据持久化，Python AI服务承担复杂的NLP计算任务，通过标准化接口与后端通信。数据层采用图数据库与向量数据库的混合存储架构，实现知识结构化与语义检索的协同优化。

7. 开发里程碑规划
项目开发采用增量迭代策略，分阶段交付核心功能，确保项目的可控性与可维护性。每个阶段通过技术验证与功能闭环，降低技术风险。
Phase 1: 基础架构搭建（P0 - 优先级最高）
项目启动期的核心任务，奠定系统开发的坚实基础。
1. 后端框架初始化：项目目录结构规划、依赖管理配置、路由模块搭建
2. 数据持久化层：SQLite 数据库连接池配置、ORM 映射关系定义、数据模型设计
3. 身份认证体系：JWT Token 认证机制、用户注册与登录逻辑、权限中间件实现
4. 前端框架集成：Vue 3 项目初始化、API 请求封装、路由配置与页面导航
5. 基础交互层：API 接口统一错误处理、数据状态管理、页面组件结构化
 
Phase 2: 知识资源管理（P0 - 优先级最高）
构建文档处理管道，为知识图谱构建提供数据基础。
1. 文件上传组件：多文件批量上传、分片传输机制、上传进度实时反馈
2. 文档管理模块：文档列表展示、状态机管理（待处理/处理中/已完成/失败）、文件元数据维护
3. 删除机制：软删除与物理删除策略、关联数据清理、资源回收
 
Phase 3: 知识图谱构建（P0 - 优先级最高）
核心技术实现阶段，实现从非结构化文档到结构化知识的转化。
1. AI 服务框架：FastAPI 应用初始化、异步任务调度、服务监控机制
2. 文档解析引擎：PDF/DOCX 文档解析、文本清洗、格式保留
3. 智能分块算法：语义分块策略、块长度控制、关联度计算
4. NLP 处理管道：命名实体识别（NER）、关系抽取模型、属性提取
5. 图数据库集成：Neo4j 连接池配置、Cypher 查询构建、节点-关系映射
6. 向量检索集成：文本向量化模型、FAISS 索引构建、相似度计算
 
Phase 4: 语义问答服务（P0 - 优先级最高）
实现核心价值功能，提供知识问答服务。
1. 查询接口：自然语言预处理、查询意图识别、参数校验
2. 向量检索服务：相似度匹配、Top-K 结果获取、结果重排序
3. 图查询服务：路径查询、邻居节点扩展、知识推理
4. 答案合成引擎：多源答案融合、置信度评分、引用来源标记
5. 知识推荐：相关节点推荐、相似问题检索
 
Phase 5: 智能题库生成（P1 - 优先级高）
自动化测评功能，降低教师工作负担。
1. 题目生成引擎：基于知识图谱的题目模板、认知层次映射、生成策略配置
2. 题目编辑界面：题目编辑器、答案解析编写、预览与校验
3. 审核发布流程：题目审核机制、发布队列管理、版本控制
 
Phase 6: 学习效果评估（P1 - 优先级高）
学习诊断与反馈机制，支持个性化学习。
1. 在线评测系统：题目展示、答案提交、自动评分规则
2. 反馈系统：即时反馈、详细解析、知识点关联分析
3. 学习统计模块：知识点掌握度矩阵、题型正确率统计、学习曲线可视化
4. 错题分析：错题集管理、薄弱知识点识别、复习建议生成
 
Phase 7: 管理控制台（P2 - 优先级中）
系统运维与管理功能，保障系统稳定运行。
1. 课程管理：课程创建、章节编排、资源配属
2. 用户管理：用户列表、权限分配、账户封禁
3. 系统配置：参数配置、监控告警、日志审计
 
8. 验收标准与质量度量
项目验收需从功能完整性、系统性能与安全性三个维度进行综合评估，确保交付成果符合预期质量标准。
功能验收标准
●文档上传服务：支持多格式文件上传，状态机流转正确，文件元数据完整
●知识图谱构建：实体抽取准确率≥85%，关系抽取准确率≥80%，图谱覆盖率≥90%
●向量检索性能：向量空间构建正确，相似度计算准确，检索结果相关度高
●语义问答服务：查询响应正常，答案相关性高，引用来源准确
●题目生成功能：生成题目格式规范，认知层次覆盖完整，答案准确无误
●答题评测系统：评分逻辑正确，反馈信息详尽，学习统计准确
性能验收标准
文档处理与实时检索服务是性能关键路径，需满足高并发与低延迟要求。

单文档处理吞吐量：≥5个文档/小时（10MB/文档）
问答查询响应时间：P95 < 2秒，P99 < 5秒
并发支持能力：≥100 并发用户，系统资源利用率≤80%

系统稳定性：7×24小时运行无崩溃，平均故障间隔≥100小时
安全性验收标准
系统需防范未授权访问与数据泄露风险，保障用户隐私与系统安全。
 
身份认证：未登录用户访问受保护资源自动跳转登录
接口鉴权：所有 API 请求携带有效 JWT Token，Token 过期自动失效
文件上传防护：文件类型白名单校验，恶意文件识别，大小限制≥10MB
敏感信息保护：数据库密码、API 密钥不硬编码，敏感配置存储加密
输入验证：SQL 注入、XSS 攻击防护，参数校验机制完善
