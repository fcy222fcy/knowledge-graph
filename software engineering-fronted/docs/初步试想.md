
```text
学生上传文档
       ↓
 文档预处理 & 分块
       ↓
 文本向量化 / 知识图谱构建
       ↓
 用户提问 / 自动生成问题
       ↓
 系统匹配知识图谱 → 给出答案或评估
       ↓
 学生回答 → 系统评分 / 反馈
```


```text
                ┌───────────────────────┐
                │        前端（Vue）      │
                │  - 页面展示            │
                │  - 用户提问            │
                │  - 文档上传            │
                └──────────┬────────────┘
                           │ HTTP请求
                           ↓
                ┌───────────────────────┐
                │     后端（Go + Gin）   │
                │  - 路由处理            │
                │  - 业务逻辑            │
                │  - 调用AI服务          │
                └───────┬───────┬───────┘
                        │       │
        HTTP调用        │       │ 数据存储
                        ↓       ↓
        ┌───────────────────────┐     ┌───────────────────────┐
        │     AI服务（Python）   │     │      数据存储层        │
        │-----------------------│     │-----------------------│
        │ 1. 文档解析           │     │ Neo4j（知识图谱）      │
        │ 2. NLP处理            │     │ - 实体                │
        │ 3. 向量生成           │     │ - 关系                │
        │ 4. 问题生成           │     │                       │
        └──────────┬────────────┘     │ FAISS（向量数据库）   │
                   │                  │ - 文本向量            │
                   ↓                  └───────────────────────┘
        ┌───────────────────────┐
        │   返回处理结果（JSON） │
        └───────────────────────┘
```
整体流程就是，前端 vue，后端用go处理请求，python处理文本，nlp分析文本，最后两个存储，Neo4j是存的实体之间的关系，向量库表示他们之间的关联性有多大

## 📋 本地开发配置

### 端口分配
- 前端 (Vue)：5173
- 后端 (Go + Gin)：8080
- AI服务：5000
- Neo4j：7474 (Web界面) / 7687 (Bolt协议)

### 技术栈
- 前端：Vue 3 + Vite + Element Plus
- 后端：Go 1.21+ + Gin + GORM + SQLite
- AI服务：Python 3.10+ + spaCy + FAISS + Neo4j
- 存储：Neo4j + FAISS

### 环境搭建步骤
1. 安装 Go 1.21+
2. 安装 Python 3.10+
3. 安装 Node.js 18+
4. 安装 Neo4j Community Edition
5. 启动前端：`cd frontend-vue-app && npm install && npm run dev`
6. 启动后端：`go run cmd/server/main.go`
7. 启动AI服务：`cd python-ai-service && pip install -r requirements.txt && python app.py`

详细需求分析请查看：[需求分析报告.md](./需求分析报告.md)

## 📊 架构说明

### 分层架构

```
┌─────────────────────────────────────┐
│         前端层 (Vue 3)               │
│  - 组件化设计                        │
│  - 响应式数据                        │
│  - 路由管理                          │
└─────────────────────────────────────┘
                 ↑
                 │ REST API
                 ↓
┌─────────────────────────────────────┐
│         后端层 (Go + Gin)            │
│  ┌─────────────────────────────┐   │
│  │  API层 (Controller)          │   │
│  │  - 路由处理                  │   │
│  │  - 参数验证                  │   │
│  │  - 响应格式化                │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │  服务层 (Service)            │   │
│  │  - 业务逻辑                  │   │
│  │  - 流程编排                  │   │
│  │  - 事务管理                  │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │  仓储层 (Repository)         │   │
│  │  - 数据访问                  │   │
│  │  - CRUD操作                  │   │
│  │  - 查询优化                  │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
                 ↑
                 │ HTTP
                 ↓
┌─────────────────────────────────────┐
│         数据层                       │
│  ┌─────────────────────────────┐   │
│  │  Neo4j (知识图谱)            │   │
│  │  - 实体节点                  │   │
│  │  - 关系边                    │   │
│  │  - 图谱查询                  │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │  FAISS (向量库)              │   │
│  │  - 文本向量                  │   │
│  │  - 相似度检索                │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

### 关键设计原则

1. **关注点分离**：API层、服务层、仓储层职责明确
2. **依赖注入**：通过构造函数注入依赖，便于测试和扩展
3. **接口抽象**：定义清晰的接口，便于实现不同存储后端
4. **错误处理**：统一的错误处理机制
5. **日志记录**：完整的日志记录，便于调试和监控
6. **性能优化**：异步处理、批量操作、缓存等
